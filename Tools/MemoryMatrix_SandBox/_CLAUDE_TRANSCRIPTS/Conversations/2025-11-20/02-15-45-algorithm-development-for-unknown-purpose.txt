- For specific use cases (like autonomous agents), yes - but these are typically built differently
- Instead of one persistent LLM, you might have:
  - A lightweight orchestrator (cheap, always-on)
  - LLM calls on-demand (expensive, bursty)

## Why TSP/Longest Path Fails

**The naive approach:**
```
LP[i, j] = max(LP[i, x] + cost(x, j))
         where x is any vertex with edge to j
```

**Problem 1: No memory of visited vertices**
- You might revisit vertex j (creates a cycle)
- "Simplicity" (no repeated vertices) requires knowing **which** vertices you've visited
- That's not just "where you are"—it's **how you got there**

**Problem 2: No inherent ordering**
- Vertices in a graph have no natural "left-to-right" sequence
- Can't evaluate "smaller problems first"
- Get stuck in infinite loops

---

## The Fix (that makes it exponential)

```
LP[i, j, S_ij] = longest path from i to j 
                 where S_ij is the SET of intermediate vertices
```

**Now it works, BUT:**
- 2^n possible sets S
- Exponential state space
- DP becomes as slow as brute force
