## Stage 1: The Plumbing

### Event Bus (`voice/pipeline/event_bus.py`)

The nervous system. Everything talks through this.

```python
# Components publish events
await event_bus.emit("speech_start", {"source": "user"})

# Other components subscribe
event_bus.subscribe("speech_start", my_handler)
```

Events: `speech_start`, `speech_end`, `interrupt`, `context_ready`, `tts_chunk_ready`, `playback_start`, `playback_done`, `virtue_updated`, `generation_done`, `generation_abort`

### Buffers (`voice/pipeline/buffers.py`)

Three buffers manage data flow:

| Buffer | What it holds | Overflow strategy |
|--------|--------------|-------------------|
| `STTBuffer` | Transcript segments | DROP_OLDEST (default) |
| `LLMBuffer` | Token stream + abort flag | Generation IDs |
| `TTSQueue` | Audio chunks | Flush on interrupt |

The key insight: each buffer can be flushed independently. When you interrupt, TTS flushes but STT keeps your new input.

### State Machine (`voice/pipeline/state_machine.py`)

Five states, validated transitions:

```
IDLE ──→ LISTENING ──→ PROCESSING ──→ SPEAKING
  ↑                                       │
  │                                       ↓
  └─────────────────────────────────── iLUNA
                                    (interrupt state)
```

The `iLUNA` state is critical. You can't go `SPEAKING → LISTENING` directly - that creates race conditions. `iLUNA` is the airlock where:
- TTS flushes
- LLM aborts
- Virtue processes the interruption
- THEN you transition to LISTENING

## Stage 4: Collision Detection

This is where it gets interesting. Eight ways things can go wrong:

### 🔇 Deaf Luna (Critical)
**Problem**: TTS keeps playing during user interrupt  
**Detection**: Speech detected while TTS active  
**Resolution**: Flush TTS immediately

### 🎭 Virtue Lag (High)
**Problem**: LLM starts before virtue state updates  
**Detection**: Context built with stale virtue timestamp  
**Resolution**: Context lock - wait for virtue before LLM starts

### 💥 Buffer Overflow (Medium)
**Problem**: User talks faster than STT processes  
**Detection**: STT buffer exceeds max size  
**Resolution**: DROP_OLDEST strategy kicks in

### ⏱️ LLM Timeout (Critical)
**Problem**: Claude takes too long, user changes topic  
**Detection**: Generation exceeds timeout threshold  
**Resolution**: Abort signal, start fresh

### 🎢 Emotional Whiplash (Medium)
**Problem**: User mood swings faster than virtue tracks  
**Detection**: Large virtue delta in short window  
**Resolution**: Smoothing caps the delta

### 🔄 Echo Feedback (High)
**Problem**: Luna hears herself through the mic  
**Detection**: STT matches recent TTS output  
**Resolution**: Clear STT buffer

### ⚡ Interrupt Cascade (High)
**Problem**: Multiple interrupts faster than recovery  
**Detection**: >N interrupts in time window  
**Resolution**: Debounce, rate limit

### 📦 Context Staleness (Medium)
**Problem**: Virtue state too old when LLM reads it  
**Detection**: Virtue timestamp older than threshold  
**Resolution**: Force refresh before generation
