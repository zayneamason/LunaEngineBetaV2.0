Ahab's asking about .venv - this is a quick question, so quick answer. He's asking what the .venv directory/file is.

.venv is the standard Python virtual environment directory. It contains:
- An isolated Python interpreter
- Installed packages (pip install goes here)
- Scripts/binaries for that environment

It's created by `python -m venv .venv` and activated with `source .venv/bin/activate`.

Quick answer, no fluff needed.`.venv` is Python's virtual environment directory.

**What's in it:**
- Isolated Python interpreter (copy/symlink)
- `site-packages/` — all your pip-installed dependencies
- `bin/` (or `Scripts/` on Windows) — activation scripts, installed CLI tools

**Why it exists:**
- Project isolation — dependencies don't bleed between projects
- Reproducibility — `pip freeze > requirements.txt` captures exact state
- No sudo — installs to project dir, not system Python

**Common workflow:**
```bash
python -m venv .venv          # create it
source .venv/bin/activate     # activate (adds to PATH)
pip install -r requirements.txt
deactivate                    # when done
```

**Should you commit it?** No — it's in `.gitignore`. Recreate from `requirements.txt` or `pyproject.toml`.

Ahab is asking a fundamental question about what a daemon is. This is a basic systems concept, so I should explain it clearly and concisely.A daemon is a background process that runs continuously without user interaction.

**The name:** From Greek mythology — a supernatural being that works in the background (not "demon"). Unix folks in the 70s thought it fit.

**Key traits:**
- **No terminal** — detached from any user session
- **Starts at boot** (or manually) and runs until stopped
- **Waits for work** — listens for signals, timers, or requests
- **Parent is PID 1** — orphaned intentionally so it survives logout

**Common examples:**
- `httpd` / `nginx` — waits for web requests
- `sshd` — waits for SSH connections
- `cron` — wakes up periodically, runs scheduled jobs
- `dockerd` — Docker's background engine

**The "-d" convention:** Daemon names often end in 'd' — `systemd`, `launchd`, `sshd`, `mysqld`.

**Your Hub Daemon specifically:**
```
You log out → Hub keeps running
You open Claude Desktop → MCP connects to Hub
You close Claude Desktop → Hub keeps running
You open Voice Luna → connects to same Hub
Memory persists across all of it
```

It's the thing that makes Luna *persistent* rather than ephemeral. Without it, every chat session would be a fresh instance with no continuity.

## The Query Language

One language that spans all index types:

```sql
-- Hybrid query: semantic + keyword + filter + graph + time decay
SELECT n.*, 
       semantic_score(n.embedding, :query_vec) as sem,
       keyword_score(n.content, 'memory architecture') as kw,
       graph_distance(n, :anchor_node) as hops,
       decay(n.created_at, half_life='30 days') as freshness
FROM nodes n
WHERE n.type = 'FACT'
  AND n.created_at > '2025-01-01'
  AND graph_reachable(n, :anchor_node, max_hops=3)
ORDER BY (sem * 0.5 + kw * 0.3 + freshness * 0.2) DESC
LIMIT 10;
```

**What's happening:**
- `semantic_score()` — vector similarity (uses HNSW index)
- `keyword_score()` — BM25 full-text (uses inverted index)
- `graph_distance()` — shortest path (uses graph index)
- `decay()` — time-based weight decay
- All fused into one query plan

## Key Features

### 1. Multi-Embedding Support

Nodes can have multiple embeddings from different models:

```sql
CREATE TABLE embeddings (
    node_id     TEXT,
    model       TEXT,      -- 'openai-3-small', 'nomic-v1.5', 'local-gte'
    vector      BLOB,
    created_at  TIMESTAMP,
    PRIMARY KEY (node_id, model)

JOIN embeddings e ON n.id = e.node_id
WHERE e.model = 'nomic-v1.5'
ORDER BY vec_distance(e.vector, :query) 
LIMIT 10;

-- Or let the system pick best available

ORDER BY semantic_score(n, :query, model_preference=['local-gte', 'nomic-v1.5'])
LIMIT 10;
```

**Why:** Swap embedding models without rebuilding. Keep old embeddings while migrating. Use different models for different query types.

---

### 2. Native Decay

Time-aware relevance built into the index:

```sql
-- Decay function is index-aware
CREATE INDEX idx_vectors_decayed ON embeddings 
USING hnsw(vector) 
WITH (decay_column='created_at', half_life='60 days');

-- Query automatically applies decay
SELECT * FROM nodes
ORDER BY semantic_score_decayed(embedding, :query)
LIMIT 10;
```

**Why:** Recent memories matter more. Old stuff fades unless reinforced.

---

### 3. Graph-Aware Vector Search

Constrain vector search by graph topology:

```sql
-- Only search nodes within 2 hops of anchor
SELECT * FROM nodes n
WHERE graph_reachable(n, :current_topic_node, max_hops=2)
ORDER BY semantic_score(n.embedding, :query)
LIMIT 10;
```

**Why:** Context matters. "Similar" means different things depending on where you are in the knowledge graph.

---

### 4. Lazy Embedding

Don't embed everything upfront:

```sql
-- Insert without embedding
INSERT INTO nodes (id, content) VALUES ('abc', 'Some text');

-- Embedding computed on first query hit, then cached
-- Or background job fills them in during idle time

SELECT * FROM nodes WHERE needs_embedding = true LIMIT 100;
```

**Why:** Embedding is expensive. Do it incrementally, prioritize hot paths.

---

### 5. Embedding Inheritance

Nodes can inherit/aggregate embeddings from children:

```sql
-- A "chapter" node's embedding is derived from its "paragraph" children
UPDATE nodes SET embedding = (
    SELECT avg_vector(e.vector)
    FROM edges ed
    JOIN embeddings e ON ed.target = e.node_id
    WHERE ed.source = nodes.id
    AND ed.type = 'contains'
)
WHERE type = 'chapter';
```

**Why:** Hierarchical content shouldn't require separate embedding strategies.

---

### 6. Portable Single File

Everything in one file that can be copied, backed up, encrypted:

```
memory.luna  (single file)
├── [header]
├── [nodes table]
├── [edges table]  
├── [embeddings table]
├── [btree indexes]
├── [fts inverted index]
├── [hnsw vector index]
├── [graph adjacency index]
└── [wal log]
```

**Why:** Luna is a file. Not Luna plus a vector service plus a graph DB.

---

### 7. Tiered Storage

Hot data in memory, warm on SSD, cold archived:

```sql
-- Mark access patterns, system manages tiers
SELECT * FROM nodes; -- promotes to hot tier

-- Explicit tier control
ALTER NODE 'old-stuff' SET TIER = 'cold';

-- Query still works, just slower
SELECT * FROM nodes WHERE tier = 'cold' AND ...;
```

**Why:** Can't keep everything in RAM on a Pi.

        self.conn.row_factory = sqlite3.Row
        self.conn.execute("PRAGMA journal_mode=WAL")
        self.conn.execute("PRAGMA foreign_keys=ON")
        
        # Load sqlite-vec extension
        self.conn.enable_load_extension(True)
        sqlite_vec.load(self.conn)
        self.conn.enable_load_extension(False)
        
        # Initialize schema
        self._init_schema()
        
        # Runtime graph (loaded from edges table)
        self.graph = nx.DiGraph()
        self._load_graph()
    
    # =========================================================================
    # SCHEMA
    # =========================================================================
    
    def _init_schema(self):
        """Create tables if they don't exist."""
        
        # Nodes
        self.conn.execute("""
            CREATE TABLE IF NOT EXISTS nodes (
                id              TEXT PRIMARY KEY,
                content         TEXT NOT NULL,
                node_type       TEXT NOT NULL,
                source          TEXT,
                created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
                updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
                accessed_at     TIMESTAMP,
                access_count    INTEGER DEFAULT 0,
                metadata        JSON
            )
        """)
        
        # FTS5 virtual table
        self.conn.execute("""
            CREATE VIRTUAL TABLE IF NOT EXISTS nodes_fts USING fts5(
                id, content, node_type,
                content='nodes',
                content_rowid='rowid',
                tokenize='porter'
            )
        """)
        
        # FTS triggers
        self.conn.executescript("""
            CREATE TRIGGER IF NOT EXISTS nodes_ai AFTER INSERT ON nodes BEGIN
                INSERT INTO nodes_fts(rowid, id, content, node_type) 
                VALUES (new.rowid, new.id, new.content, new.node_type);
            END;
            
            CREATE TRIGGER IF NOT EXISTS nodes_ad AFTER DELETE ON nodes BEGIN
                DELETE FROM nodes_fts WHERE rowid = old.rowid;
            END;
            
            CREATE TRIGGER IF NOT EXISTS nodes_au AFTER UPDATE ON nodes BEGIN
                DELETE FROM nodes_fts WHERE rowid = old.rowid;
                INSERT INTO nodes_fts(rowid, id, content, node_type) 
                VALUES (new.rowid, new.id, new.content, new.node_type);
            END;
        """)
        
        # Embeddings
        self.conn.execute("""
            CREATE TABLE IF NOT EXISTS embeddings (
                node_id         TEXT NOT NULL,
                model           TEXT NOT NULL,
                embedding       BLOB NOT NULL,
                created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
                PRIMARY KEY (node_id, model),
                FOREIGN KEY (node_id) REFERENCES nodes(id) ON DELETE CASCADE
            )
        """)
        
        # Vector index (sqlite-vec)
        self.conn.execute(f"""
            CREATE VIRTUAL TABLE IF NOT EXISTS vec_index USING vec0(
                node_id TEXT PRIMARY KEY,
                model TEXT,
                embedding float[{self.embedding_dim}]
            )
        """)
        
        # Edges
        self.conn.execute("""
            CREATE TABLE IF NOT EXISTS edges (
                source          TEXT NOT NULL,
                target          TEXT NOT NULL,
                edge_type       TEXT NOT NULL,
                weight          REAL DEFAULT 1.0,
                created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
                metadata        JSON,
                PRIMARY KEY (source, target, edge_type),
                FOREIGN KEY (source) REFERENCES nodes(id) ON DELETE CASCADE,
                FOREIGN KEY (target) REFERENCES nodes(id) ON DELETE CASCADE
            )
        """)
        
        # Indexes
        self.conn.executescript("""
            CREATE INDEX IF NOT EXISTS idx_nodes_type ON nodes(node_type);
            CREATE INDEX IF NOT EXISTS idx_nodes_created ON nodes(created_at);
            CREATE INDEX IF NOT EXISTS idx_nodes_accessed ON nodes(accessed_at);
            CREATE INDEX IF NOT EXISTS idx_embeddings_model ON embeddings(model);
            CREATE INDEX IF NOT EXISTS idx_edges_source ON edges(source);
            CREATE INDEX IF NOT EXISTS idx_edges_target ON edges(target);
        """)
        
        self.conn.commit()
    
    def _load_graph(self):
        """Load edges into NetworkX graph."""
        self.graph.clear()
        
        # Add all nodes
        for row in self.conn.execute("SELECT id FROM nodes"):
            self.graph.add_node(row["id"])
        
        # Add all edges
        for row in self.conn.execute(
            "SELECT source, target, edge_type, weight FROM edges"
        ):
            self.graph.add_edge(
                row["source"], 
                row["target"],
                edge_type=row["edge_type"],
                weight=row["weight"]
            )
    
    # =========================================================================
    # NODE OPERATIONS
    # =========================================================================
    
    def add_node(
        self,
        content: str,
        node_type: str,
        id: Optional[str] = None,
        source: Optional[str] = None,
        embedding: Optional[np.ndarray] = None,
        embedding_model: str = DEFAULT_MODEL,
        metadata: Optional[dict] = None,
    ) -> Node:
        """
        Add a node to the store.
        
        If embedding is provided, indexes it immediately.
        If not, node can be embedded later (lazy embedding).
        """
        # Generate ID if not provided
        if id is None:
            id = self._generate_id(content)
        
        now = datetime.now().isoformat()
        meta_json = json.dumps(metadata or {})
        
        self.conn.execute("""
            INSERT INTO nodes (id, content, node_type, source, created_at, updated_at, metadata)
            VALUES (?, ?, ?, ?, ?, ?, ?)
            ON CONFLICT(id) DO UPDATE SET
                content = excluded.content,
                node_type = excluded.node_type,
                updated_at = excluded.updated_at,
                metadata = excluded.metadata
        """, (id, content, node_type, source, now, now, meta_json))
        
        # Add to graph
        self.graph.add_node(id)
        
        # Add embedding if provided
        if embedding is not None:
            self._add_embedding(id, embedding, embedding_model)
        
        self.conn.commit()
        
        return Node(
            id=id,
            content=content,
            node_type=node_type,
            source=source,
            created_at=datetime.fromisoformat(now),
            metadata=metadata or {}
        )
    
    def _add_embedding(self, node_id: str, embedding: np.ndarray, model: str):
        """Add or update embedding for a node."""
        emb_bytes = embedding.astype(np.float32).tobytes()
        
        # Store in embeddings table
        self.conn.execute("""
            INSERT INTO embeddings (node_id, model, embedding)
            VALUES (?, ?, ?)
            ON CONFLICT(node_id, model) DO UPDATE SET
                embedding = excluded.embedding,
                created_at = CURRENT_TIMESTAMP
        """, (node_id, model, emb_bytes))
        
        # Update vector index
        self.conn.execute("""
            INSERT INTO vec_index (node_id, model, embedding)
            VALUES (?, ?, ?)
            ON CONFLICT(node_id) DO UPDATE SET
                embedding = excluded.embedding
        """, (node_id, model, embedding.tolist()))
    
    def get_node(self, node_id: str) -> Optional[Node]:
        """Get a node by ID, updating access tracking."""
        row = self.conn.execute(
            "SELECT * FROM nodes WHERE id = ?", (node_id,)
        ).fetchone()
        
        if not row:
            return None
        
        # Update access tracking
        self.conn.execute("""
            UPDATE nodes 
            SET accessed_at = CURRENT_TIMESTAMP, access_count = access_count + 1
            WHERE id = ?
        """, (node_id,))
        self.conn.commit()
        
        return self._row_to_node(row)
    
    def _row_to_node(self, row: sqlite3.Row) -> Node:
        """Convert database row to Node object."""
        return Node(
            id=row["id"],
            content=row["content"],
            node_type=row["node_type"],
            source=row["source"],
            created_at=self._parse_timestamp(row["created_at"]),
            updated_at=self._parse_timestamp(row["updated_at"]),
            accessed_at=self._parse_timestamp(row["accessed_at"]),
            access_count=row["access_count"] or 0,
            metadata=json.loads(row["metadata"]) if row["metadata"] else {}
        )
    
    # =========================================================================
    # EDGE OPERATIONS
    # =========================================================================
    
    def add_edge(
        self,
        source: str,
        target: str,
        edge_type: str,
        weight: float = 1.0,
        metadata: Optional[dict] = None
    ) -> Edge:
        """Add an edge between nodes."""
        meta_json = json.dumps(metadata or {})
        
        self.conn.execute("""
            INSERT INTO edges (source, target, edge_type, weight, metadata)
            VALUES (?, ?, ?, ?, ?)
            ON CONFLICT(source, target, edge_type) DO UPDATE SET
                weight = excluded.weight,
                metadata = excluded.metadata
        """, (source, target, edge_type, weight, meta_json))
        
        # Update runtime graph
        self.graph.add_edge(source, target, edge_type=edge_type, weight=weight)
        
        self.conn.commit()
        
        return Edge(source=source, target=target, edge_type=edge_type, weight=weight)
    
    # =========================================================================
    # QUERY ENGINE
    # =========================================================================
    
    def query(
        self,
        # Search inputs (use any combination)
        semantic: Optional[str | np.ndarray] = None,
        keywords: Optional[str] = None,
        graph_anchor: Optional[str] = None,
        
        # Filters
        node_types: Optional[list[str]] = None,
        since: Optional[datetime] = None,
        max_hops: int = 3,
        
        # Scoring
        decay_half_life_days: float = 60.0,
        semantic_weight: float = 0.5,
        keyword_weight: float = 0.3,
        freshness_weight: float = 0.2,
        
        # Fusion
        fusion: Literal["rrf", "weighted"] = "rrf",
        
        # Output
        limit: int = 10,
        embedding_model: str = DEFAULT_MODEL,
        embed_fn: Optional[callable] = None,  # For embedding query strings
    ) -> QueryResult:
        """
        Unified query across all backends.
        
        Args:
            semantic: Query vector or string (requires embed_fn if string)
            keywords: Keywords for FTS5 search
            graph_anchor: Node ID to constrain by graph distance
            node_types: Filter by node types
            since: Filter by creation date
            max_hops: Max graph distance from anchor
            decay_half_life_days: Half-life for freshness decay
            semantic_weight: Weight for semantic score in fusion
            keyword_weight: Weight for keyword score in fusion
            freshness_weight: Weight for freshness score in fusion
            fusion: 'rrf' (reciprocal rank fusion) or 'weighted' (score combination)
            limit: Max results to return
            embedding_model: Which embedding model to use for vector search
            embed_fn: Function to embed query string (required if semantic is str)
        
        Returns:
            QueryResult with nodes and metadata
        """
        start_time = time.time()
        backends_used = []
        
        # Candidate sets from each backend: {node_id: score}
        semantic_scores = {}
        keyword_scores = {}
        graph_distances = {}
        
        # 1. Semantic search
        if semantic is not None:
            backends_used.append("semantic")
            
            # Convert string to vector if needed
            if isinstance(semantic, str):
                if embed_fn is None:
                    raise ValueError("embed_fn required when semantic query is a string")
                query_vec = embed_fn(semantic)
            else:
                query_vec = semantic
            
            semantic_scores = self._semantic_search(
                query_vec, 
                limit=limit * 3,  # oversample for fusion
                model=embedding_model
            )
        
        # 2. Keyword search
        if keywords:
            backends_used.append("keyword")
            keyword_scores = self._keyword_search(keywords, limit=limit * 3)
        
        # 3. Graph constraint
        if graph_anchor:
            backends_used.append("graph")
            graph_distances = self._graph_distances(graph_anchor, max_hops)
        
        # 4. Get candidate node IDs
        all_candidates = set(semantic_scores.keys()) | set(keyword_scores.keys())
        
        # If graph anchor specified, filter to only graph-reachable nodes
        if graph_anchor and graph_distances:
            all_candidates &= set(graph_distances.keys())
        
        # If no search terms, get recent nodes
        if not all_candidates and not semantic and not keywords:
            all_candidates = self._get_recent_node_ids(limit * 3)
        
        # 5. Apply filters
        filtered_candidates = self._apply_filters(
            all_candidates, 
            node_types=node_types,
            since=since
        )
        
        # 6. Load nodes and compute freshness
        nodes = []
        for node_id in filtered_candidates:
            node = self._get_node_without_access_update(node_id)
            if node:
                # Attach scores
                node.semantic_score = semantic_scores.get(node_id, 0.0)
                node.keyword_score = keyword_scores.get(node_id, 0.0)
                node.graph_distance = graph_distances.get(node_id)
                node.freshness = self._compute_freshness(
                    node.created_at, 
                    decay_half_life_days
                )
                nodes.append(node)
        
        # 7. Fusion scoring
        if fusion == "rrf":
            nodes = self._rrf_fusion(nodes, semantic_weight, keyword_weight, freshness_weight)
        else:
            nodes = self._weighted_fusion(nodes, semantic_weight, keyword_weight, freshness_weight)
        
        # 8. Sort and limit
        nodes.sort(key=lambda n: n.combined_score or 0, reverse=True)
        nodes = nodes[:limit]
        
        query_time_ms = (time.time() - start_time) * 1000
        
        return QueryResult(
            nodes=nodes,
            total_candidates=len(filtered_candidates),
            query_time_ms=query_time_ms,
            backends_used=backends_used
        )
    
    # =========================================================================
    # SEARCH BACKENDS
    # =========================================================================
    
    def _semantic_search(
        self, 
        query_vec: np.ndarray, 
        limit: int,
        model: str
    ) -> dict[str, float]:
        """Vector similarity search using sqlite-vec."""
        results = {}
        
        rows = self.conn.execute("""
            SELECT node_id, distance
            FROM vec_index
            WHERE model = ?
              AND embedding MATCH ?
            ORDER BY distance
            LIMIT ?
        """, (model, query_vec.tolist(), limit)).fetchall()
        
        for row in rows:
            # Convert distance to similarity (assumes cosine distance)
            similarity = 1.0 - row["distance"]
            results[row["node_id"]] = max(0.0, similarity)
        
        return results
    
    def _keyword_search(self, query: str, limit: int) -> dict[str, float]:
        """BM25 full-text search using FTS5."""
        results = {}
        
        rows = self.conn.execute("""
            SELECT id, bm25(nodes_fts) as score
            FROM nodes_fts
            WHERE nodes_fts MATCH ?
            ORDER BY score
            LIMIT ?
        """, (query, limit)).fetchall()
        
        # BM25 scores are negative (lower is better), normalize
        if rows:
            min_score = min(r["score"] for r in rows)
            max_score = max(r["score"] for r in rows)
            score_range = max_score - min_score if max_score != min_score else 1.0
            
            for row in rows:
                # Normalize to 0-1 (inverted because BM25 is negative)
                normalized = 1.0 - (row["score"] - min_score) / score_range
                results[row["id"]] = normalized
        
        return results
    
    def _graph_distances(self, anchor: str, max_hops: int) -> dict[str, int]:
        """Get all nodes within max_hops of anchor."""
        if anchor not in self.graph:
            return {}
        
        distances = {}
        
        # BFS from anchor
        try:
            lengths = nx.single_source_shortest_path_length(
                self.graph, 
                anchor, 
                cutoff=max_hops
            )
            distances = dict(lengths)
        except nx.NetworkXError:
            pass
        
        # Also check reverse direction (undirected traversal)
        try:
            reverse_lengths = nx.single_source_shortest_path_length(
                self.graph.reverse(), 
                anchor, 
                cutoff=max_hops
            )
            for node, dist in reverse_lengths.items():
                if node not in distances or dist < distances[node]:
                    distances[node] = dist
        except nx.NetworkXError:
            pass
        
        return distances
    
    # =========================================================================
    # FUSION
    # =========================================================================
    
    def _rrf_fusion(
        self, 
        nodes: list[Node],
        sem_weight: float,
        kw_weight: float,
        fresh_weight: float
    ) -> list[Node]:
        """
        Reciprocal Rank Fusion.
        
        Combines rankings from multiple sources using:
        RRF(d) = Σ 1 / (k + rank(d))
        
        More robust than score combination when scores aren't comparable.
        """
        k = 60  # Standard RRF constant
        
        # Create rankings for each signal
        def rank_by(nodes: list[Node], key) -> dict[str, int]:
            sorted_nodes = sorted(nodes, key=key, reverse=True)
            return {n.id: i + 1 for i, n in enumerate(sorted_nodes)}
        
        sem_ranks = rank_by(nodes, lambda n: n.semantic_score or 0)
        kw_ranks = rank_by(nodes, lambda n: n.keyword_score or 0)
        fresh_ranks = rank_by(nodes, lambda n: n.freshness or 0)
        
        # Compute RRF score
        for node in nodes:
            rrf_score = 0.0
            
            if sem_weight > 0 and node.semantic_score:
                rrf_score += sem_weight / (k + sem_ranks.get(node.id, len(nodes)))
            
            if kw_weight > 0 and node.keyword_score:
                rrf_score += kw_weight / (k + kw_ranks.get(node.id, len(nodes)))
            
            if fresh_weight > 0 and node.freshness:
                rrf_score += fresh_weight / (k + fresh_ranks.get(node.id, len(nodes)))
            
            node.combined_score = rrf_score
        
        return nodes
    
    def _weighted_fusion(
        self,
        nodes: list[Node],
        sem_weight: float,
        kw_weight: float,
        fresh_weight: float
    ) -> list[Node]:
        """Simple weighted score combination."""
        for node in nodes:
            node.combined_score = (
                sem_weight * (node.semantic_score or 0) +
                kw_weight * (node.keyword_score or 0) +
                fresh_weight * (node.freshness or 0)
            )
        return nodes
    
    # =========================================================================
    # DECAY
    # =========================================================================
    
    def _compute_freshness(
        self, 
        created_at: Optional[datetime],
        half_life_days: float
    ) -> float:
        """
        Exponential decay based on age.
        
        Returns 1.0 for brand new, 0.5 at half_life_days, approaches 0 over time.
        """
        if created_at is None:
            return 0.5  # Unknown age gets neutral score
        
        age_days = (datetime.now() - created_at).total_seconds() / 86400
        decay = math.exp(-math.log(2) * age_days / half_life_days)
        
        return decay
    
    # =========================================================================
    # HELPERS
    # =========================================================================
    
    def _apply_filters(
        self,
        candidates: set[str],
        node_types: Optional[list[str]] = None,
        since: Optional[datetime] = None,
    ) -> set[str]:
        """Apply SQL filters to candidate set."""
        if not candidates:
            return set()
        
        query = "SELECT id FROM nodes WHERE id IN ({})".format(
            ",".join("?" * len(candidates))
        )
        params = list(candidates)
        
        if node_types:
            query += " AND node_type IN ({})".format(
                ",".join("?" * len(node_types))
            )
            params.extend(node_types)
        
        if since:
            query += " AND created_at > ?"
            params.append(since.isoformat())
        
        rows = self.conn.execute(query, params).fetchall()
        return {row["id"] for row in rows}
    
    def _get_recent_node_ids(self, limit: int) -> set[str]:
        """Get most recently created node IDs."""
        rows = self.conn.execute(
            "SELECT id FROM nodes ORDER BY created_at DESC LIMIT ?",
            (limit,)
        ).fetchall()
        return {row["id"] for row in rows}
    
    def _get_node_without_access_update(self, node_id: str) -> Optional[Node]:
        """Get node without updating access tracking."""
        row = self.conn.execute(
            "SELECT * FROM nodes WHERE id = ?", (node_id,)
        ).fetchone()
        return self._row_to_node(row) if row else None
    
    def _generate_id(self, content: str) -> str:
        """Generate deterministic ID from content."""
        return hashlib.sha256(content.encode()).hexdigest()[:16]
    
    def _parse_timestamp(self, ts: Optional[str]) -> Optional[datetime]:
        """Parse timestamp string to datetime."""
        if ts is None:
            return None
        try:
            return datetime.fromisoformat(ts)
        except ValueError:
            return None
    
    # =========================================================================
    # MAINTENANCE
    # =========================================================================
    
    def rebuild_graph(self):
        """Reload graph from edges table."""
        self._load_graph()
    
    def vacuum(self):
        """Optimize database."""
        self.conn.execute("VACUUM")
    
    def checkpoint(self):
        """Force WAL checkpoint."""
        self.conn.execute("PRAGMA wal_checkpoint(TRUNCATE)")
    
    def stats(self) -> dict:
        """Get store statistics."""
        return {
            "nodes": self.conn.execute("SELECT COUNT(*) FROM nodes").fetchone()[0],
            "edges": self.conn.execute("SELECT COUNT(*) FROM edges").fetchone()[0],
            "embeddings": self.conn.execute("SELECT COUNT(*) FROM embeddings").fetchone()[0],
            "graph_nodes": self.graph.number_of_nodes(),
            "graph_edges": self.graph.number_of_edges(),
        }
    
    def close(self):
        """Close database connection."""
        self.checkpoint()
        self.conn.close()

## What It Does

Adds a new virtual table type that stores vectors and supports similarity search:

```sql
-- Create a vector table
CREATE VIRTUAL TABLE my_vectors USING vec0(
    id TEXT PRIMARY KEY,
    embedding float[384]  -- 384-dimensional vectors
);

-- Insert vectors
INSERT INTO my_vectors (id, embedding) 
VALUES ('doc1', '[0.1, 0.2, 0.3, ...]');

-- Search by similarity
SELECT id, distance
FROM my_vectors
WHERE embedding MATCH '[0.15, 0.25, 0.35, ...]'
ORDER BY distance
LIMIT 10;
```

## How It Works

```
┌─────────────────────────────────────────────────────────────────┐
│                         SQLite                                  │
│                                                                 │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────────┐ │
│  │   Regular   │  │    FTS5     │  │      sqlite-vec         │ │
│  │   Tables    │  │  (text)     │  │      (vectors)          │ │
│  │             │  │             │  │                         │ │
│  │  B-tree     │  │  Inverted   │  │  Flat or HNSW index     │ │
│  │  index      │  │  index      │  │  for ANN search         │ │
│  └─────────────┘  └─────────────┘  └─────────────────────────┘ │
│                                                                 │
│                    All in one .db file                          │
└─────────────────────────────────────────────────────────────────┘
```

It's the same pattern as FTS5 (full-text search) — a loadable extension that adds specialized indexing to SQLite.

---

## The Key Benefit

**No separate vector database.**

| Without sqlite-vec | With sqlite-vec |
|--------------------|-----------------|
| SQLite for metadata | SQLite for everything |
| + FAISS for vectors | |
| + sync logic between them | |
| + two files to manage | One file |

---

## Installation

```bash
pip install sqlite-vec
```

```python
import sqlite3
import sqlite_vec

db = sqlite3.connect("my.db")
db.enable_load_extension(True)
sqlite_vec.load(db)
db.enable_load_extension(False)

# Now you can use vec0 virtual tables
```

---

## Index Types

sqlite-vec supports two modes:

**1. Flat (brute force)**
```sql
-- Exact search, O(n) but accurate
CREATE VIRTUAL TABLE vecs USING vec0(
    embedding float[384]
);
```

**2. IVF (approximate)**
```sql
-- Faster search, slightly less accurate
CREATE VIRTUAL TABLE vecs USING vec0(
    embedding float[384],
    +nlist=100  -- number of clusters
);
```

No HNSW yet (that's in the roadmap), but IVF handles moderate scale fine.

---

## Limitations

| Aspect | sqlite-vec | FAISS |
|--------|-----------|-------|
| Speed (pure ANN) | Good | Better |
| Index types | Flat, IVF | Flat, IVF, HNSW, PQ, etc. |
| Filtering | Native SQL | Manual post-filter |
| Maturity | Newer (2024) | Battle-tested |
| Integration | SQLite native | Separate system |
| Scale | ~1M vectors | 100M+ vectors |

---

## Why It Fits Luna

1. **Single file** — vectors in `memory_matrix.db` with everything else
2. **SQL filtering** — `WHERE node_type = 'FACT' AND embedding MATCH ...`
3. **Transactions** — vector updates atomic with node updates
4. **Portable** — copy one file, Luna moves
5. **No sync bugs** — can't have vectors orphaned from nodes

For 776 nodes (or even 100K), sqlite-vec is plenty. You'd only outgrow it at serious scale.

---

## Quick Comparison

```python
# FAISS approach (what you have now)
ids, distances = faiss_index.search(query_vec, k=10)
node_ids = [id_map[i] for i in ids]
nodes = db.execute("SELECT * FROM nodes WHERE id IN (?)", node_ids)
# Then manually filter by type, date, etc. in Python

# sqlite-vec approach
nodes = db.execute("""
    SELECT n.*, v.distance
    FROM nodes n
    JOIN vec_index v ON n.id = v.node_id
    WHERE n.node_type = 'FACT'
      AND n.created_at > '2025-01-01'
      AND v.embedding MATCH ?
    ORDER BY v.distance
    LIMIT 10
""", [query_vec])
# Filtering happens in the query, one round trip
```

The second version is cleaner and eliminates the sync problem entirely.
