## The Agent Loop (Claude Code Style)

```python
class LunaAgentLoop:
    """
    Autonomous execution loop.
    Runs until goal achieved or max_iterations hit.
    """
    
    def __init__(self, orchestrator: Orchestrator, max_iterations: int = 50):
        self.orchestrator = orchestrator
        self.max_iterations = max_iterations
        self.working_context = WorkingContext()
        self.task_queue = TaskQueue()
    
    async def run(self, goal: str) -> AgentResult:
        """Main agent loop — observe, think, act, repeat."""
        
        # Initial planning
        plan = await self.orchestrator.decompose(goal)
        self.task_queue.add_all(plan.tasks)
        
        iteration = 0
        while not self.is_complete() and iteration < self.max_iterations:
            iteration += 1
            
            # OBSERVE: What's the current state?
            observation = await self.observe()
            self.working_context.update(observation)
            
            # THINK: What should we do next?
            next_action = await self.orchestrator.decide(
                goal=goal,
                context=self.working_context,
                pending_tasks=self.task_queue,
            )
            
            if next_action.type == ActionType.COMPLETE:
                break
            
            # ACT: Execute the action
            result = await self.execute(next_action)
            
            # UPDATE: Record what happened
            self.working_context.add_result(next_action, result)
            
            # STREAM: Keep user informed
            await self.stream_progress(next_action, result)
        
        # SYNTHESIZE: Create final response
        return await self.orchestrator.synthesize(
            goal=goal,
            context=self.working_context,
        )
    
    async def execute(self, action: Action) -> ActionResult:
        """Route action to appropriate worker."""
        match action.worker:
            case "librarian":
                return await self.librarian.execute(action)
            case "oven":
                return await self.oven.execute(action)
            case "tooler":
                return await self.tooler.execute(action)
            case "coder":
                return await self.coder.execute(action)
            case _:
                return await self.orchestrator.execute_direct(action)
```

## Parallel Execution (Claude Flow Style)

```python
class LunaSwarm:
    """
    Parallel worker execution.
    Multiple agents working simultaneously.
    """
    
    async def execute_parallel(
        self, 
        tasks: list[Task],
        max_concurrent: int = 5
    ) -> list[TaskResult]:
        """Run multiple tasks in parallel."""
        
        semaphore = asyncio.Semaphore(max_concurrent)
        
        async def run_with_limit(task: Task) -> TaskResult:
            async with semaphore:
                worker = self.get_worker(task.worker_type)
                return await worker.execute(task)
        
        results = await asyncio.gather(
            *[run_with_limit(t) for t in tasks],
            return_exceptions=True
        )
        
        return results
    
    async def fan_out_fan_in(
        self,
        goal: str,
        perspectives: list[str]
    ) -> SynthesizedResult:
        """
        Fan out: Multiple workers research same goal from different angles.
        Fan in: Synthesize their findings.
        """
        
        # Fan out — parallel research
        tasks = [
            Task(
                type="research",
                goal=goal,
                perspective=p,
                worker_type="oven"
            )
            for p in perspectives
        ]
        
        results = await self.execute_parallel(tasks)
        
        # Fan in — synthesize
        return await self.orchestrator.synthesize_perspectives(results)
```

## The Key Concepts

### 1. **Revolving Context (Not Static)**

Context isn't a flat list — it's a **rotating structure** where:
- Items have *positions* (S1-S5 slots)
- Items have *relevance weights* that change over time
- Core (Luna's identity) is always center
- Items orbit — can move closer or further based on relevance

### 2. **Multiple Queues, One Manager**

Not a single input buffer. Multiple specialized queues:
- Conversation queue (user messages, responses)
- Memory queue (retrieved context)
- Tool queue (results from tool executions)
- Task queue (pending work)

Queue Manager merges with priority.

### 3. **Context Borrowing**

Luna, Scribe, and Librarian can **share context**:
- Scribe processing conversation? Luna can see what Scribe sees.
- Librarian retrieving memory? That context is available to Luna.
- Not isolated silos — coordinated awareness.

### 4. **Token Budget (Z)**

Fixed budget. Context pool can't exceed Z tokens.
- As new items come in, old items get pushed out
- Relevance weight determines what stays
- Core (Luna identity) always stays

### 5. **Context Timeout / TTL**

Items age out:
- Fresh item: high relevance, stays close to core
- Aging item: relevance decays
- Timed out: evicted from context

This is **attention decay** implemented as architecture.

### 6. **Task Manager**

Separate from conversation flow:
- Tasks queue up
- Get executed
- Results feed back into context

This enables the agent loop — tasks aren't conversation, they're work.
