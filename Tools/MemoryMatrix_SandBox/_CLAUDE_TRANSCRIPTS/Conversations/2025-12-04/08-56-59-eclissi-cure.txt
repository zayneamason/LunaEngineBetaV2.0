## Core Concept: What smolagents Actually Is

It's NOT a full orchestration framework. It's a **minimal agent runner** - about 1,000 lines of code. The philosophy is "just enough abstraction to be useful, not so much you don't understand what's happening."

## Two Agent Types

### 1. **CodeAgent** (Primary - what you want)
- Agent writes **Python code** to accomplish tasks
- Tools are exposed as Python functions
- Natural composability: loops, conditions, function nesting
- Example:
```python
from smolagents import CodeAgent, InferenceClientModel, WebSearchTool

model = InferenceClientModel()  # or local model
agent = CodeAgent(
    tools=[WebSearchTool()],
    model=model
)
result = agent.run("How many seconds for a leopard to run across Pont des Arts?")
```

**What the agent actually does:**
```python
# Agent generates this code:
search_result = web_search("length of Pont des Arts")
# processes result
leopard_speed = 58  # km/h from knowledge
# does math
final_answer(result)
```

### 2. **ToolCallingAgent** (Alternative)
- Agent writes JSON/text tool calls
- More traditional agent pattern
- Less flexible but more predictable

## The Architecture Pattern

```python
# Basic structure
agent = CodeAgent(
    tools=[list_of_tools],      # What it can do
    model=your_model,            # What thinks
    system_prompt="...",         # How it behaves
    add_base_tools=True          # Includes python_interpreter, etc.
)

# Execute
result = agent.run("task description")
```
