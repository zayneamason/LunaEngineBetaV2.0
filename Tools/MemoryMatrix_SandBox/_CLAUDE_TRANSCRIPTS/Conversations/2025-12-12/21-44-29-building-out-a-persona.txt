### 1. **The Identity Problem**
ResponseGenerator's system prompt has this line:
```

```

This is *excellent* philosophy. But the implementation depends entirely on Memory Matrix having Luna's identity stored as memories. That's fragile — Luna's core identity shouldn't be emergent from memory retrieval, it should be *foundational*.

### 2. **Personality vs Identity**
The current system conflates two things:
- **Personality** = How Luna responds (playfulness, warmth, technical_depth) — should be emergent, dynamic
- **Identity** = Who Luna *is* (relationship with Ahab, history, core values) — should be stable, foundational

PersonalityComputer handles personality. But identity just comes from... whatever smart_fetch returns? That's backwards.

### 3. **The Virtue System Gap**
In your user preferences, there's this:
```
- Kernel (Luna's core identity)
- Virtues (personality from memory/virtue/current.json)
```

But I don't see `kernel` or `virtues` referenced anywhere in PersonaCore/ResponseGenerator. That's a big miss — those should be the *foundation* that memory and personality layer on top of.
