```python
        memory_context = response.get("memory_context", [])
        if memory_context and params.auto_fetch:
            result_parts.append("## Relevant Memories")
            for ctx in memory_context:
                result_parts.append(ctx)
            result_parts.append("")

        # Debug info if available
        debug = response.get("debug")
        if debug:
            result_parts.append("## Debug")
            result_parts.append(f"*Processing: {debug.get('api_latency_ms', 0):.0f}ms*")

        return "\n".join(result_parts)

    except Exception as e:
        # Fallback: if Hub/PersonaCore unavailable, use local state machine only
        try:
            sm = get_state_machine()
            state = sm.process_message(params.message)
            
            return f"""## Luna State (FALLBACK - Hub unavailable)
**Meta**: {state.meta.value}
**Mode**: {state.mode.value if state.mode else 'N/A'}
**Full Path**: {state.full_path}

⚠️ PersonaCore unavailable: {str(e)}
Memory context and personality not loaded.
"""
        except Exception as fallback_error:
            return f"Error during context detection: {str(e)} (fallback also failed: {fallback_error})"
```

### What This Fix Does:

1. **Routes through `/persona/process`** instead of `/memory/smart-fetch`
2. **Gets full PersonaCore pipeline**: state → consciousness → personality → memory → enrichment
3. **Graceful fallback**: If Hub is down, still returns basic state machine info
4. **Desktop-appropriate**: `generate_response: False` means no LLM call, just context enrichment
