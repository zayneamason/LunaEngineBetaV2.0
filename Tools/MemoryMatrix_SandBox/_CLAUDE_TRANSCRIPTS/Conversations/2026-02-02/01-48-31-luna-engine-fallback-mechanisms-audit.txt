Alright man, here's the deal with director.py. It's got some classic symptoms.

## The Smells

**1. God Object Alert**
The Director is doing too many jobs:
- State machine management
- LLM routing decisions  
- Context assembly
- Response generation
- Error handling
- Provider selection

Bible says the Director owns *orchestration*. Everything else should be delegated to other actors. Right now it's a 600-line monolith trying to be the whole brain.

**2. No Fallback Chain = Constitutional Violation**

The Bible explicitly states:
> "Constitutional Principle: Luna ALWAYS responds. Graceful degradation, never silence."

Current behavior when Claude fails:
```python
except Exception as e:
    return "I'm having trouble processing that right now."
```

That's not graceful degradation. That's silence with extra steps. The spec says:
- Claude unavailable → Warm tier handles
- Warm tier unavailable → Hot tier handles
- Hot tier unavailable → Voice Actor plays error message

You've got the tiers defined but no cascade logic.

**3. Heuristic vs Trained Routing**

Current `_should_delegate()` uses hardcoded heuristics:
- Complexity threshold (0.15)
- Keyword matching ("delegation signals")

Bible says the Director should be **trained** to know when to delegate via `<REQ_CLAUDE>` token emission. The routing should emerge from fine-tuning, not if-else chains.

This is the "solve the stated problem vs solve the real problem" gap. The stated problem is "route to the right model." The real problem is "teach Luna to know her own limits."

## What's Right

- LLM Registry exists - good abstraction point
- Actor mailbox pattern is there
- Complexity estimation exists (even if the threshold is arbitrary)

## Architectural Fix

The file needs to be split along responsibility lines:

```
DirectorActor (orchestration only)
    ├── delegates to → RouteSelector (routing decisions)
    ├── delegates to → ContextAssembler (prompt building)
    ├── delegates to → FallbackChain (provider cascade)
    └── delegates to → ResponseValidator (output quality)
```

The routing logic should eventually be a LoRA capability, not a code path.
