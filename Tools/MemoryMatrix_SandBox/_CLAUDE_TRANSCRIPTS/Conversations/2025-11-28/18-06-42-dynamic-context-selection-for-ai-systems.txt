## For Sub-problem 2: Within-Cluster Aggregation

### Option A: Simple Arithmetic Mean
```
cluster_score = Σ(fader_values) / n
```
- **Time complexity**: O(n) where n = faders in cluster
- **Space complexity**: O(1)
- **Best when**: All faders contribute equally
- **Fails when**: Some faders should dominate
- **Key parameters**: None

### Option B: Weighted Mean
```
cluster_score = Σ(weight_i × fader_i) / Σ(weight_i)
```
- **Time complexity**: O(n)
- **Space complexity**: O(n) for weights
- **Best when**: Domain knowledge says some faders matter more
- **Fails when**: Weights are miscalibrated
- **Key parameters**: weight vector (requires tuning)

### Option C: Geometric Mean
```
cluster_score = (Π fader_i)^(1/n)
```
- **Time complexity**: O(n)
- **Space complexity**: O(1)
- **Best when**: You want to penalize zeros heavily (all faders must contribute)
- **Fails when**: Any fader is 0 (kills entire score)
- **Key parameters**: None

### Option D: Max-Weighted Blend
```
cluster_score = α × max(faders) + (1-α) × mean(faders)
```
- **Time complexity**: O(n)
- **Space complexity**: O(1)
- **Best when**: Lead fader should dominate but others still matter
- **Fails when**: α is wrong
- **Key parameters**: α (blend factor, typically 0.3-0.5)

### Option E: RMS (Root Mean Square)
```
cluster_score = √(Σ(fader_i²) / n)
```
- **Time complexity**: O(n)
- **Space complexity**: O(1)
- **Best when**: Higher values should disproportionately influence score
- **Fails when**: You want linear relationship
- **Key parameters**: None

**Recommendation**: **Option D (Max-Weighted Blend)** with α=0.4 — this naturally emphasizes the lead fader while preserving influence of supporting faders. Interpretable and tunable.

---

## For Sub-problem 3: Lead Detection

### Option A: Simple Argmax
```
lead = argmax(faders_in_cluster)
```
- **Time complexity**: O(n)
- **Best when**: Clear winner exists
- **Fails when**: Values are close → lead flip-flops on small changes

### Option B: Argmax with Hysteresis
```
if new_max > current_lead_value + hysteresis_threshold:
    lead = new_max_fader
else:
    lead = current_lead  # sticky
```
- **Time complexity**: O(n)
- **Best when**: Stability matters, small changes shouldn't flip lead
- **Fails when**: Threshold too high → lead never changes
- **Key parameters**: hysteresis_threshold (typically 5-10 on 0-100 scale)

### Option C: Weighted Argmax
```
effective_value = fader_value × domain_weight
lead = argmax(effective_values)
```
- **Time complexity**: O(n)
- **Best when**: Some faders are "more important" as leads
- **Fails when**: Weights feel arbitrary
- **Key parameters**: domain_weight vector

**Recommendation**: **Option B (Argmax with Hysteresis)** — prevents annoying flip-flopping while remaining simple and interpretable. Threshold of 7 (on 0-100 scale) works well.

---

## For Sub-problem 4: Profile Classification

### Option A: Threshold-Based Rules
```
if cluster_score < 33: level = "low"
elif cluster_score < 66: level = "medium"  
else: level = "high"

profile = f"{lead}_{level}"
```
- **Time complexity**: O(1)
- **Best when**: Clear cutoffs exist
- **Fails when**: Boundary cases feel wrong
- **Key parameters**: threshold values

### Option B: Fuzzy Membership
```
membership_low = triangular(score, 0, 0, 50)
membership_med = triangular(score, 25, 50, 75)
membership_high = triangular(score, 50, 100, 100)

profile = weighted_label(memberships)
```
- **Time complexity**: O(k) where k = number of levels
- **Best when**: Smooth transitions between labels
- **Fails when**: Fuzzy output is confusing to users
- **Key parameters**: membership function shapes

### Option C: Lookup Table
```
profile_map = {
    ("game_design", "high"): "game_design_expert",
    ("game_design", "medium"): "game_design_practitioner",
    ("coordination", "high"): "coordination_authority",
    ...
}
profile = profile_map[(lead, quantize(score))]
```
- **Time complexity**: O(1) hash lookup
- **Best when**: You want full control over labels
- **Fails when**: Combinatorial explosion of cases
- **Key parameters**: The entire map (design decision)

**Recommendation**: **Option C (Lookup Table)** with **Option A thresholds** for quantization. This gives you full control over the vocabulary while keeping computation trivial.

# === IMPLEMENTATION ===

```javascript
// Full implementation: ~50 lines, <1ms execution

const CLUSTERS = {
  expertise: ['game_design', 'fabrication', 'coordination'],
  tone: ['authority', 'specificity', 'formality'],
  energy: ['energy']
};

const BLEND_ALPHA = 0.4;
const HYSTERESIS = 7;
const THRESHOLDS = [33, 66];

// State for hysteresis
let currentLeads = { expertise: null, tone: null };

function computeClusterScore(faders, clusterFaders, alpha = BLEND_ALPHA) {
  const values = clusterFaders.map(f => faders[f]);
  const maxVal = Math.max(...values);
  const meanVal = values.reduce((a, b) => a + b, 0) / values.length;
  return alpha * maxVal + (1 - alpha) * meanVal;
}

function detectLead(faders, clusterFaders, clusterName) {
  const values = clusterFaders.map(f => ({ name: f, value: faders[f] }));
  const maxFader = values.reduce((a, b) => a.value > b.value ? a : b);
  
  const currentLead = currentLeads[clusterName];
  if (currentLead) {
    const currentValue = faders[currentLead];
    if (maxFader.value > currentValue + HYSTERESIS) {
      currentLeads[clusterName] = maxFader.name;
      return maxFader.name;
    }
    return currentLead;
  }
  
  currentLeads[clusterName] = maxFader.name;
  return maxFader.name;
}

function quantize(score, thresholds = THRESHOLDS) {
  if (score < thresholds[0]) return 'low';
  if (score < thresholds[1]) return 'medium';
  return 'high';
}

function getProfile(lead, level, clusterName) {
  // Lookup table - customize these labels
  const profiles = {
    expertise: {
      game_design: { low: 'observer', medium: 'practitioner', high: 'authority' },
      fabrication: { low: 'appreciator', medium: 'maker', high: 'craftsman' },
      coordination: { low: 'participant', medium: 'facilitator', high: 'orchestrator' }
    },
    tone: {
      authority: { low: 'humble', medium: 'confident', high: 'commanding' },
      specificity: { low: 'conceptual', medium: 'balanced', high: 'tactical' },
      formality: { low: 'casual', medium: 'professional', high: 'formal' }
    },
    energy: {
      energy: { low: 'calm', medium: 'engaged', high: 'intense' }
    }
  };
  
  return profiles[clusterName]?.[lead]?.[level] || `${lead}_${level}`;
}

function clusterFaders(faders) {
  const result = {};
  
  for (const [clusterName, clusterFaders] of Object.entries(CLUSTERS)) {
    const score = Math.round(computeClusterScore(faders, clusterFaders));
    const lead = clusterFaders.length > 1 
      ? detectLead(faders, clusterFaders, clusterName)
      : clusterFaders[0];
    const level = quantize(score);
    const profile = getProfile(lead, level, clusterName);
    
    result[clusterName] = { score, lead, level, profile };
  }
  
  return result;
}

function explainCluster(faders, clusterName) {
  const clusterFaders = CLUSTERS[clusterName];
  const totalWeight = clusterFaders.length;
  
  return clusterFaders.map(f => ({
    fader: f,
    value: faders[f],
    contribution: Math.round((faders[f] / totalWeight) * 100) / 100,
    isLead: f === currentLeads[clusterName]
  }));
}
```

# === TESTING STRATEGY ===

**Unit tests:**
1. Score calculation matches expected for known inputs

2. Compare outputs — clusters should produce equivalent behavioral result
3. Adjust α and thresholds if outputs diverge

# === GOTCHAS ===

1. **Hysteresis state**: Lead detection requires remembering previous state. In stateless contexts, either pass previous lead in, or accept flip-flopping.

2. **Threshold sensitivity**: Values at exactly 33 or 66 will classify differently than 32 or 67. Consider fuzzy boundaries if this causes UX issues.

3. **Profile vocabulary**: The lookup table grows combinatorially. For 3 leads × 3 levels = 9 profiles per cluster. Keep vocabulary manageable.

4. **Equal faders**: When all faders in a cluster are equal, lead detection is arbitrary. Consider a "balanced" label for this case.
