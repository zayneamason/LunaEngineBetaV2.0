## A* Pathfinding - The Basics

**What it does:** Finds the shortest path between two points on a graph/grid.

**Core idea:** Combines two things:
1. **g(n)** = actual cost from start to current node
2. **h(n)** = estimated cost from current node to goal (the "heuristic")
3. **f(n) = g(n) + h(n)** = total estimated cost

Always explore the node with the lowest f(n) first.

---

## The Algorithm

```
1. Create two lists:
   - OPEN (nodes to evaluate)
   - CLOSED (nodes already evaluated)

2. Add start node to OPEN

3. Loop:
   a. Pick node with lowest f(n) from OPEN → call it CURRENT
   b. Move CURRENT to CLOSED
   
   c. If CURRENT is the goal → done, retrace path
   
   d. For each NEIGHBOR of CURRENT:
      - If NEIGHBOR is in CLOSED or unwalkable → skip
      - Calculate tentative g = CURRENT.g + distance(CURRENT, NEIGHBOR)
      
      - If NEIGHBOR not in OPEN → add it
      - Else if tentative g >= NEIGHBOR.g → skip (not a better path)
      
      - Update NEIGHBOR:
        - parent = CURRENT
        - g = tentative g
        - h = heuristic(NEIGHBOR, goal)
        - f = g + h

4. If OPEN is empty and goal not reached → no path exists
```

---

## The Heuristic (h)

This is where it gets interesting. Common options:

| Heuristic | Formula | Best For |
|-----------|---------|----------|
| **Manhattan** | \|x1-x2\| + \|y1-y2\| | 4-directional grids |
| **Euclidean** | √((x1-x2)² + (y1-y2)²) | Any-angle movement |
| **Chebyshev** | max(\|x1-x2\|, \|y1-y2\|) | 8-directional grids |

**Rule:** Heuristic must be *admissible* (never overestimate actual cost) to guarantee shortest path.

---

## Simple Python Implementation

```python
import heapq

def astar(grid, start, goal):
    rows, cols = len(grid), len(grid[0])
    
    # Priority queue: (f_score, node)
    open_set = [(0, start)]
    came_from = {}
    
    g_score = {start: 0}
    f_score = {start: heuristic(start, goal)}
    
    while open_set:
        current = heapq.heappop(open_set)[1]
        
        if current == goal:
            return reconstruct_path(came_from, current)
        
        for neighbor in get_neighbors(current, grid):
            tentative_g = g_score[current] + 1  # assuming cost = 1
            
            if tentative_g < g_score.get(neighbor, float('inf')):
                came_from[neighbor] = current
                g_score[neighbor] = tentative_g
                f_score[neighbor] = tentative_g + heuristic(neighbor, goal)
                heapq.heappush(open_set, (f_score[neighbor], neighbor))
    
    return None  # No path

def heuristic(a, b):
    return abs(a[0] - b[0]) + abs(a[1] - b[1])  # Manhattan

def get_neighbors(node, grid):
    directions = [(0,1), (1,0), (0,-1), (-1,0)]
    neighbors = []
    for d in directions:
        new = (node[0] + d[0], node[1] + d[1])
        if 0 <= new[0] < len(grid) and 0 <= new[1] < len(grid[0]):
            if grid[new[0]][new[1]] != 1:  # 1 = wall
                neighbors.append(new)
    return neighbors

def reconstruct_path(came_from, current):
    path = [current]
    while current in came_from:
        current = came_from[current]
        path.append(current)
    return path[::-1]
```

## Pathfinding as Musical Structure: A Synthesis

### 1. The Heuristic as Expectation System

The research showed that emotion in music comes from **violation of expectation**—Leonard Meyer's core insight. A* works the same way:

- **h(n)** = the listener's *prediction* of where the music is going
- **g(n)** = what *actually happened* so far
- **f(n)** = the felt sense of "how far are we from resolution?"

When h(n) accurately predicts actual cost, the path feels "consonant"—satisfying, expected. When reality diverges from heuristic (a wall appears, terrain cost spikes), that's a **deceptive cadence**. The algorithm has to recalculate, just like a listener reorients when Mozart lands on vi instead of I.

**Implication:** A* with a perfect heuristic is like a completely predictable piece of music—efficient but boring. The *interesting* paths happen when the heuristic gets surprised.

---

### 2. Hierarchical Pathfinding = Phrase Nesting

Classical music uses **2-4-8-16 bar nesting**—containers within containers. Beethoven's genius was knowing when to break the container (3-bar phrases, 10-bar phrases) for emotional effect.

Hierarchical Pathfinding (HPA*) works identically:

```
Level 3:  [=======REGION A=======]  [=======REGION B=======]
Level 2:  [==Zone 1==][==Zone 2==]  [==Zone 3==][==Zone 4==]
Level 1:  [cells][cells][cells]...  [cells][cells][cells]...
```

The algorithm plans at the macro level first (which regions?), then refines (which zones?), then executes (which cells?).

**The Beethoven Move:** What if we deliberately break hierarchical symmetry? A path that spends 8 steps in Region A, then 10 in Region B, then 6 in Region C creates **asymmetric phrasing**—forward momentum through unfulfilled expectation. The path *feels* more dynamic than a symmetric 8-8-8.

Beethoven's Fifth places its recapitulation at bar 372 of 601—**0.618**, the golden ratio. The emotional climax lands at φ.

### 8. False Recapitulation = The Fake Goal

Beethoven's Eroica has a **false horn entrance**—the theme appears to return, but it's premature, in the wrong key. Maximum tension.

In game design, this is the **visible-but-unreachable objective**. The player's internal heuristic screams "go there!" but the path requires going away first. That's the false recapitulation—emotional tension through expectation violation.

**Deliberate design:** Place geometry that makes h(n) misleadingly low, forcing recalculation and the emotional beat of "I thought I was close."

### 9. The Coda as Path Resolution

Beethoven invented the **structural coda**—not a brief sign-off but a full quarter of the movement. The Fifth Symphony's coda is 129 bars, nearly equal to exposition/development/recap.

```
Traditional path:
[====journey====][goal]

Beethoven path:
[==journey==][====approach/arrival====]
```

The final 38% of the path should have its own dramatic structure—not just "almost there" repetition but genuine development. Slow down, add flourishes, make arrival feel earned.
