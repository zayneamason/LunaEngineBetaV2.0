```javascript
// Per-sentence decision at boundary
for (let i = 0; i < n; i++) {
  // CONSTRAINT: Can't switch mid-sentence
  if (!isNewSentence && lastAssignment) {
    assignments.push(lastAssignment);
    continue;
  }
  
  // Calculate current ratio
  const currentRatio = totalSpent > 0 ? costA / totalSpent : 0;
  
  // Greedy decision: assign to whoever is further from target
  const remainingCostA = targetCostA - costA;
  const remainingCostB = targetCostB - costB;

  }
```

**Algorithm:** At each sentence boundary, look at current ratio vs target α. Assign to whichever LLM is "behind" on its quota. Local optimization only — no lookahead.

**Failure mode:** Can get trapped. If early sentences happen to be expensive and you greedily assign them all to A, you might overshoot and can't recover.

## The Dynamic Programming Algorithm

This is the interesting one. Let me trace through it:

### State Definition
```javascript
// M[s][costA] = { waste, lastAssignment, prevCostA }
// s = sentence index (0 to numSentences)
// costA = cumulative cost allocated to LLM A
```

**State space:** For each sentence position, track all possible cumulative costs to A. This is a 2D table: `sentences × possible_cost_sums`.

### Transition Logic
```javascript
for (let s = 1; s <= numSentences; s++) {
  const sentenceCost = sentenceGroups[s-1].totalCost;
  
  for (let costA = 0; costA <= totalCost; costA++) {
    
    // Option 1: Assign sentence s to LLM A
    if (costA >= sentenceCost) {
      const prevCostA = costA - sentenceCost;
      // Calculate penalties...
      const switchPenalty = (lastAssignment === 'B') ? switchCostRate * totalCost : 0;
      const deviationPenalty = Math.abs(currentRatio - targetAlpha) * totalCost;
      // Update if better than current best
    }
    
    // Option 2: Assign sentence s to LLM B (costA stays same)
    // Similar logic...
  }
}
```

**Recurrence:**
```
M[s][costA] = min(
  M[s-1][costA - sentenceCost] + switchPenalty_A + deviationPenalty,  // assign to A
  M[s-1][costA] + switchPenalty_B + deviationPenalty                   // assign to B
)
```

### Cost Function Components

1. **Switch Penalty:** `switchCostRate * totalCost` if switching from A↔B
   - Models context-switching overhead
   - Currently 5% of total budget per switch

2. **Deviation Penalty:** `|currentRatio - targetAlpha| * totalCost`
   - How far current allocation is from target
   - Penalized at each step, not just at end

### Backtracking
```javascript
// Find best final state
for (let costA = 0; costA <= totalCost; costA++) {
  if (M[numSentences][costA].waste < bestWaste) {
    bestWaste = M[numSentences][costA].waste;
    bestCostA = costA;
  }
}

// Reconstruct path backwards
for (let s = numSentences; s > 0; s--) {
  sentenceAssignments.unshift(M[s][currentCostA].lastAssignment);
  currentCostA = M[s][currentCostA].prevCostA;
}
```
