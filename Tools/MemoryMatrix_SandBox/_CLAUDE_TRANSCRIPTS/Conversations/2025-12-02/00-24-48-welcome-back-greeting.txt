### Architecture Decision: QDockWidget vs Multi-Window

**Going with QDockWidget (Option A).** Here's why:

1. **Shared event loop** — All panels update from the same `VirtueEngine` and `EventBus` instances. No IPC overhead.
2. **Atomic state** — When virtue changes, left panel (sliders), center (debug stream), and right (thought stream) all update from the same source of truth.
3. **User can still detach** — QDockWidget supports floating, so if someone wants the timeline as a separate window, they just drag it out.
4. **Simpler process model** — One Python process, one event bus, no file-watching hacks.

**Trade-off acknowledged:** Heavier single window. But M1 can handle it, and the alternative (coordinating 3-4 separate processes) adds latency and sync bugs.

```python
class VirtueSliderWidget(QFrame):
    """Single virtue with slider, zone indicator, felt sense."""
    
    virtue_changed = pyqtSignal(str, float)  # virtue_name, new_value
    
    def __init__(self, virtue_name: str, virtue_data: dict):
        # virtue_data from VIRTUE_DATA[virtue_name]
        self.virtue = virtue_name
        self.color = virtue_data['color']
        self.zones = virtue_data['zones']
        
    # Layout:
    # ┌─────────────────────────────────────────┐
    # │ ● Curiosity              [Eager] +25   │
    # │ ├──────────────●────────────────────┤  │
    # │ Disengaged    Genuine         Eager    │
    # │ "Buzzing. Hungry. Pulled forward."     │
    # └─────────────────────────────────────────┘
    
    def set_value(self, value: float):
        """Called when virtue updates from engine (don't emit signal)."""
        self._updating = True
        self.slider.setValue(int(value))
        self._update_zone_display()
        self._updating = False
        
    def _on_slider_moved(self, value: int):
        """User dragged slider — emit for engine update."""
        if not self._updating:
            self.virtue_changed.emit(self.virtue, float(value))
```

**Key detail:** Bidirectional binding. Engine updates → slider moves (no signal). User drags → signal emits → engine.apply_deltas() → coupling cascade → all sliders update.

#### 2. ThoughtStreamPanel (the hard one)

The React version uses CSS transforms and opacity. In Qt, we need custom painting.

```python
class ThoughtParticle:
    """A thought moving through the stream."""
    id: int
    text: str
    particle_type: str  # 'raw', 'thought', 'question', 'impulse', 'filtered', 'crystallized'
    virtue: Optional[str]
    x: float  # 0.0 (left) to 1.0 (right, crystallize zone)
    y: float  # vertical position
    opacity: float
    created_at: float
    
class ThoughtStreamCanvas(QWidget):
    """Custom painted thought stream visualization."""
    
    def __init__(self):
        self.particles: List[ThoughtParticle] = []
        self.timer = QTimer()
        self.timer.timeout.connect(self._tick)
        self.timer.start(33)  # ~30 FPS
        
    def add_thought(self, text: str, ptype: str, virtue: str = None):
        """Called from event bus on thought_generated."""
        particle = ThoughtParticle(
            id=len(self.particles),
            text=text,
            particle_type=ptype,
            virtue=virtue,
            x=0.1 + random.uniform(-0.05, 0.05),
            y=0.1 + len([p for p in self.particles if p.opacity > 0.1]) * 0.08,
            opacity=1.0,
            created_at=time.time(),
        )
        self.particles.append(particle)
        
    def _tick(self):
        """Animation frame — move particles, fade, repaint."""
        now = time.time()
        for p in self.particles:
            age = now - p.created_at
            
            # Drift toward crystallize zone (right side)
            if p.particle_type != 'crystallized':
                p.x += 0.002  # slow drift
                
            # Fade based on type
            if p.particle_type == 'filtered':
                p.opacity = max(0, 1.0 - age / 2.0)  # fast fade
            elif p.particle_type != 'crystallized':
                p.opacity = max(0, 1.0 - age / self.fade_speed)
                
        # Prune dead particles
        self.particles = [p for p in self.particles if p.opacity > 0.01 or p.particle_type == 'crystallized']
        self.update()  # trigger repaint
        
    def paintEvent(self, event):
        painter = QPainter(self)
        painter.setRenderHint(QPainter.RenderHint.Antialiasing)
        
        # Draw crystallize zone gradient
        # Draw particles
        for p in self.particles:
            self._draw_particle(painter, p)
            
    def _draw_particle(self, painter: QPainter, p: ThoughtParticle):
        rect = self.rect()
        x = int(p.x * rect.width())
        y = int(p.y * rect.height())
        
        color = self._get_virtue_color(p.virtue) if p.virtue else QColor('#6b7280')
        color.setAlphaF(p.opacity)
        
        # Draw virtue dot
        if p.virtue and p.particle_type != 'crystallized':
            painter.setBrush(QBrush(color))
            painter.drawEllipse(QPoint(x - 20, y), 4, 4)
            
        # Draw text
        painter.setPen(QPen(color))
        font = QFont("SF Mono", 11 if p.particle_type == 'crystallized' else 9)
        if p.particle_type == 'filtered':
            font.setStrikeOut(True)
        painter.setFont(font)
        painter.drawText(x, y, p.text)
```

**Performance note:** 30 FPS with ~20 particles max is fine. If it lags, drop to 20 FPS or batch the particle list.

```python
            data['text'],
            data['type'],
            data.get('virtue')
        ),
    ))
    
    # Pipeline state → status bar + debug
    bus.subscribe('pipeline_state', lambda data: (
```
