## DATA MODEL ARCHITECTURE

```javascript
// CAPTAIN'S PLAN
Plan {
  destination: string,
  trajectory: Phase[],
  totalBudget: number,
  allocationStrategy: string,
  penaltyStructure: PenaltyConfig,
  successCriteria: string[]
}

// PHASE STRUCTURE
Phase {
  name: string,
  turns: number,
  tokenAllocation: number,
  objective: string,
  constraints: Constraint[]
}

// COGNITIVE SYSTEM STATE
CognitiveSystem {
  id: string,
  type: 'orchestrator' | 'creative' | 'analytical',
  alpha: number,
  traits: object,  // emotions or virtues
  state: {
    currentThought: string,
    commitment: number,
    momentum: number,
    lastFeedback: object
  },
  performance: {
    tokensUsed: number,
    tokensAllocated: number,
    penaltiesIncurred: number,
    bonusesEarned: number
  }
}

// TOKEN ECONOMY STATE
Economy {
  budget: {
    total: number,
    allocated: Map<system, number>,
    spent: Map<system, number>,
    remaining: number
  },
  reserves: {
    taxPool: number,
    bonusPool: number
  },
  history: Transaction[]
}

// TRANSACTION (Penalty/Bonus)
Transaction {
  timestamp: Date,
  system: string,
  type: 'penalty' | 'bonus',
  subtype: string,  // 'tokenTax', 'excellence', etc.
  amount: number,
  reason: string,
  appealed: boolean
}

// USER VIRTUE PROFILE (Persistent)
UserProfile {
  userId: string,
  version: string,
  virtues: {
    temperance: number,
    industry: number,
    resolution: number,
    humility: number,
    tranquility: number,
    sincerity: number,
    justice: number
  },
  statistics: {
    sessionsCompleted: number,
    totalTokensUsed: number,
    averageDepth: number,
    synthesisQualityAvg: number
  },
  metadata: {
    firstSession: Date,
    lastSession: Date,
    longestDialogue: number
  }
}
```

```

   ↓
2. App.jsx loads UserProfile from localStorage
   ↓
3. Captain.receivePlan(prompt, userProfile.virtues)
   ↓
4. Captain.createExecutionPlan()

   ↓
5. Orchestrator.executePlan(plan)
   ↓
6. FOR EACH TURN:
   a. Orchestrator.selectNextSpeaker()

   c. TokenMonitor.track(system, usage)

   e. VirtueEngine.constrainBehavior()

   g. Orchestrator.reportToCaptain(turnData)
   h. Captain.monitorAdherence(turnData)
   ↓
7. Captain.checkSuccessCriteria()
   ↓
8. Captain.directSynthesis()
   ↓
9. VirtueEngine.calculateShifts(sessionData)
   ↓
10. UserProfile.update(shifts)
    ↓
11. UserProfile.save(localStorage)
```

## AMENDMENT SYSTEM

```python
class Captain:
    def __init__(self):
        self.amendment_cooldown = 5  # rounds between amendments
        self.rounds_since_amendment = 5  # starts ready
        self.pending_amendment = None
    
    def propose_amendment(self, variable, new_value, reason):
        """Queue an amendment - doesn't execute yet"""
        self.pending_amendment = {
            'variable': variable,
            'new_value': new_value,
            'reason': reason,
            'proposed_at': current_round
        }
        return "Amendment queued. Executable in {n} rounds."
    
    def can_amend(self):
        return self.rounds_since_amendment >= self.amendment_cooldown
    
    def execute_amendment(self, ship):
        """Actually change Ship's variable"""
        if not self.can_amend():
            return f"Cannot amend. {self.amendment_cooldown - self.rounds_since_amendment} rounds remaining."
        
        if not self.pending_amendment:
            return "No amendment pending."
        
        # Execute the change
        ship.set_variable(
            self.pending_amendment['variable'],
            self.pending_amendment['new_value']
        )
        
        # Reset cooldown
        self.rounds_since_amendment = 0
        self.pending_amendment = None
        
        return "Amendment executed. Next amendment available in 5 rounds."
    
    def on_round_complete(self):
        """Called each round"""
        self.rounds_since_amendment += 1
```
