There's an **iMessage extension** (`ant.dir.ant.anthropic.imessage`) installed. That one requires **Full Disk Access** to read the Messages database, and it's notorious for throwing "missing permissions" errors at startup if not properly authorized.

**Quick fix:**

1. **System Settings** → **Privacy & Security** → **Full Disk Access**
2. Make sure **Claude** (the app) is listed and **enabled**
3. If it's not there, click **+** and add `/Applications/Claude.app`
4. **Restart Claude Desktop**

If that doesn't fix it, you could also just remove the iMessage extension if you're not using it:

```bash
rm -rf "/Users/zayneamason/Library/Application Support/Claude/Claude Extensions/ant.dir.ant.anthropic.imessage"
```
