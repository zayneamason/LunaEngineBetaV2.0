### Module 2: CRUCIBLE (Synthetic Generation)

Takes Personality Atoms + Scenario Templates → Training Pairs

**Key Insight:** You need SCENARIO DIVERSITY, not just prompt diversity.

```
Scenario Categories:
├── Technical Help (coding, debugging, explaining)

Each scenario gets run through a "temperature curve" — some responses should be warmer, some cooler, to prevent the LoRA from learning a flat personality.

## Core Data Model

```sql
-- Personality being trained
CREATE TABLE personalities (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    trait_matrix JSON,      -- voice atoms, beliefs, tics
    created_at TIMESTAMP,
    source_material_hash TEXT  -- for reproducibility
);

-- Generated training pairs
CREATE TABLE training_turns (
    id TEXT PRIMARY KEY,
    personality_id TEXT REFERENCES personalities(id),
    scenario_type TEXT,      -- 'technical', 'emotional', 'casual', etc.
    instruction TEXT,        -- the prompt
    response TEXT,           -- the generated response
    temperature REAL,        -- what temp was used
    quality_score REAL,      -- post-hoc validation score
    flagged BOOLEAN DEFAULT FALSE,  -- manual review flag
    created_at TIMESTAMP
);

-- Track what source material was ingested
CREATE TABLE source_chunks (
    id TEXT PRIMARY KEY,
    personality_id TEXT REFERENCES personalities(id),
    content TEXT,
    extracted_traits JSON,   -- what Ben-pattern found here
    chunk_type TEXT          -- 'chat', 'journal', 'description'
);
```

---
{chunk}
---

Extract:
1. VOICE PATTERNS: Recurring stylistic choices (formatting, punctuation, action markers)
2. VERBAL TICS: Words/phrases used frequently
3. EMOTIONAL BASELINE: Default mood and volatility level

Output as JSON. Be specific. Quote examples from the text.

## 4. LOCKSMITH — Weighting Module

This is where the lock-in coefficient comes in.

**The Lock-In Equation (from Memory Matrix):**

```
lock_in = base_lock_in + (retrieval_bonus × retrieval_count) + (reinforcement_bonus × reinforcement_count)

Where:
- base_lock_in: Initial quality score (0.15 - 0.85)
- retrieval_bonus: 0.02 per retrieval
- reinforcement_bonus: 0.05 per manual reinforcement
- Clamped to [0.15, 0.95]
```

**Applied to Training Data:**

```python
@dataclass
class ForgeLockin:
    """Lock-in coefficient for training examples."""
    
    base_quality: float          # From voice analysis, source quality
    retrieval_bonus: float       # Referenced by other examples during synthesis
    reinforcement_bonus: float   # Manually marked as "gold"
    
    @property
    def coefficient(self) -> float:
        raw = self.base_quality + self.retrieval_bonus + self.reinforcement_bonus
        return max(0.15, min(0.95, raw))
    
    @property
    def tier(self) -> str:
        if self.coefficient >= 0.75:
            return "gold"
        elif self.coefficient >= 0.50:
            return "silver"
        else:
            return "bronze"
```

**How Lock-In Affects Training:**

```python
def compute_example_weight(candidate: TrainingCandidate) -> float:
    """
    High lock-in examples get:
    1. Duplicated in dataset (2-3x for gold)
    2. Higher loss weight during training
    3. Priority in validation set
    """
    lockin = candidate.lock_in.coefficient
    
    if lockin >= 0.75:
        return 3.0   # Gold: triple weight
    elif lockin >= 0.50:
        return 1.5   # Silver: 1.5x weight
    else:
        return 1.0   # Bronze: normal weight
```

**Lock-In Signals for Training Data:**

```python
def compute_base_quality(candidate: TrainingCandidate) -> float:
    """Score based on voice analysis."""
    score = 0.5  # baseline
    
    # Positive signals
    if has_voice_markers(candidate, "first_person"):
        score += 0.1
    if has_voice_markers(candidate, "warmth_words"):
        score += 0.1
    if has_voice_markers(candidate, "relationship"):
        score += 0.1
    if candidate.source == "journal":
        score += 0.15  # journals are gold
    
    # Negative signals
    if has_anti_patterns(candidate, "generic_ai"):
        score -= 0.3
    if has_anti_patterns(candidate, "corporate"):
        score -= 0.15
    if candidate.response_length > 300 and candidate.interaction_type == "greeting":
        score -= 0.2  # too long for type
    
    return max(0.15, min(0.85, score))
```

## 5. ANVIL — Export Module

Hammers everything into final training format.

**Export Pipeline:**

```python
def forge_dataset(
    candidates: list[TrainingCandidate],
    target_count: int,
    train_split: float = 0.9
) -> tuple[Path, Path]:
    """
    Final export with:
    1. Lock-in weighted duplication
    2. System prompt variation applied
    3. Train/val split
    4. JSONL format
    """
    
    # Weight and duplicate based on lock-in
    weighted = []
    for c in candidates:
        weight = compute_example_weight(c)
        copies = int(weight)  # 1, 2, or 3 copies
        for _ in range(copies):
            weighted.append(apply_system_prompt_variant(c))
    
    # Shuffle and split
    random.shuffle(weighted)
    split_idx = int(len(weighted) * train_split)
    train = weighted[:split_idx]
    val = weighted[split_idx:]
    
    # Export
    train_path = export_jsonl(train, "luna_dataset_train.jsonl")
    val_path = export_jsonl(val, "luna_dataset_val.jsonl")
    
    return train_path, val_path
```

## DATA MODELS

```python
# src/persona_forge/engine/models.py

from pydantic import BaseModel, Field
from enum import Enum
from datetime import datetime
from typing import Optional

class InteractionType(str, Enum):
    GREETING = "greeting"
    ACKNOWLEDGMENT = "acknowledgment"
    SHORT_EXCHANGE = "short_exchange"
    CONTEXT_RECALL = "context_recall"
    EMOTIONAL_PRESENCE = "emotional_presence"
    DELEGATION_TRIGGER = "delegation_trigger"
    REFLECTION = "reflection"
    TECHNICAL = "technical"

class QualityTier(str, Enum):
    GOLD = "gold"
    SILVER = "silver"
    BRONZE = "bronze"

class SourceType(str, Enum):
    JOURNAL = "journal"
    SESSION = "session"
    MATRIX = "matrix"
    INSIGHT = "insight"
    SYNTHETIC = "synthetic"
    MANUAL = "manual"

class LockIn(BaseModel):
    """Lock-in coefficient for training example weighting."""
    base_quality: float = Field(ge=0.0, le=1.0)
    retrieval_bonus: float = Field(default=0.0, ge=0.0)
    reinforcement_bonus: float = Field(default=0.0, ge=0.0)
    
    @property
    def coefficient(self) -> float:
        raw = self.base_quality + self.retrieval_bonus + self.reinforcement_bonus
        return max(0.15, min(0.95, raw))
    
    @property
    def tier(self) -> QualityTier:
        if self.coefficient >= 0.75:
            return QualityTier.GOLD
        elif self.coefficient >= 0.50:
            return QualityTier.SILVER
        return QualityTier.BRONZE

class TrainingExample(BaseModel):
    """Single training example with full metadata."""
    id: str
    
    # Core training data
    system_prompt: str
    user_message: str
    assistant_response: str
    
    # Metadata
    source_type: SourceType
    source_file: Optional[str] = None
    interaction_type: InteractionType
    created_at: datetime = Field(default_factory=datetime.now)
    
    # Metrics
    response_word_count: int = 0
    user_word_count: int = 0
    
    # Quality
    lock_in: LockIn
    voice_markers: dict[str, bool] = Field(default_factory=dict)
    anti_patterns: dict[str, bool] = Field(default_factory=dict)
    
    def compute_word_counts(self) -> None:
        self.response_word_count = len(self.assistant_response.split())
        self.user_word_count = len(self.user_message.split())

class ResponseLength(str, Enum):
    SHORT = "short"      # < 50 words
    MEDIUM = "medium"    # 50-150 words

### Assayer (Analysis)

```python
# src/persona_forge/engine/assayer.py

import re
from collections import Counter
from .models import (
    TrainingExample, DatasetAssay, TargetProfile,
    InteractionType, ResponseLength, QualityTier,
    DIRECTOR_PROFILE
)

class Assayer:
    """Analyzes training dataset against target profile."""
    
    VOICE_MARKERS = {
        "first_person": r"\b(I|I'm|I've|I'd|my|me)\b",
        "warmth_words": r"\b(honestly|actually|you know|yeah|cool|nice|hey)\b",
        "uncertainty": r"\b(maybe|probably|I think|not sure|might|could be)\b",
        "relationship": r"\b(we|we've|we're|our|together|Ahab)\b",
        "inside_refs": r"\b(Dude|Ben|fart|symphony|sovereignty|Memory Matrix)\b",
    }
    
    ANTI_PATTERNS = {
        "generic_ai": r"\b(I am an AI|as an AI|language model|Alibaba|Qwen|created by)\b",
        "corporate": r"\b(I'd be happy to|certainly|absolutely|assist you|help you with)\b",
        "hedging": r"\b(I cannot|I'm not able|I don't have the ability|I'm unable)\b",
    }
    
    def __init__(self, target_profile: TargetProfile = DIRECTOR_PROFILE):
        self.profile = target_profile
    
    def analyze(self, examples: list[TrainingExample]) -> DatasetAssay:
        """Full dataset analysis."""
        
        # Compute distributions
        type_dist = self._compute_type_distribution(examples)
        length_dist = self._compute_length_distribution(examples)
        source_dist = self._compute_source_distribution(examples)
        tier_dist = self._compute_tier_distribution(examples)
        
        # Voice analysis
        voice_rates = self._compute_voice_marker_rates(examples)
        anti_rates = self._compute_anti_pattern_rates(examples)
        
        # Lock-in stats
        coefficients = [e.lock_in.coefficient for e in examples]
        avg_lockin = sum(coefficients) / len(coefficients) if coefficients else 0
        
        # Compute gaps
        gaps = self._compute_gaps(type_dist, len(examples))
        synthesis_targets = self._compute_synthesis_targets(gaps)
        
        return DatasetAssay(
            total_examples=len(examples),
            interaction_type_dist=type_dist,
            response_length_dist=length_dist,
            source_type_dist=source_dist,
            quality_tier_dist=tier_dist,
            voice_marker_rates=voice_rates,
            anti_pattern_rates=anti_rates,
            avg_lock_in=avg_lockin,
            lock_in_std=self._std(coefficients),
            gaps=gaps,
            synthesis_targets=synthesis_targets,
        )
    
    def _compute_type_distribution(self, examples: list[TrainingExample]) -> dict:
        counter = Counter(e.interaction_type for e in examples)
        total = len(examples)
        return {t: counter.get(t, 0) / total for t in InteractionType}
    
    def _compute_length_distribution(self, examples: list[TrainingExample]) -> dict:
        def categorize(wc: int) -> ResponseLength:
            if wc < 50:
                return ResponseLength.SHORT
            elif wc < 150:
                return ResponseLength.MEDIUM
            return ResponseLength.LONG
        
        counter = Counter(categorize(e.response_word_count) for e in examples)
        total = len(examples)
        return {l: counter.get(l, 0) / total for l in ResponseLength}
    
    def _compute_voice_marker_rates(self, examples: list[TrainingExample]) -> dict:
        rates = {}
        for marker, pattern in self.VOICE_MARKERS.items():
            matches = sum(1 for e in examples if re.search(pattern, e.assistant_response, re.I))
            rates[marker] = matches / len(examples) if examples else 0
        return rates
    
    def _compute_anti_pattern_rates(self, examples: list[TrainingExample]) -> dict:
        rates = {}
        for pattern_name, pattern in self.ANTI_PATTERNS.items():
            matches = sum(1 for e in examples if re.search(pattern, e.assistant_response, re.I))
            rates[pattern_name] = matches / len(examples) if examples else 0
        return rates
    
    def _compute_gaps(self, actual_dist: dict, total: int) -> dict[InteractionType, int]:
        """Compute gap between actual and target counts."""
        gaps = {}
        for itype in InteractionType:
            target_pct = self.profile.interaction_types.get(itype, 0)
            actual_pct = actual_dist.get(itype, 0)
            target_count = int(self.profile.target_examples * target_pct)
            actual_count = int(total * actual_pct)
            gaps[itype] = actual_count - target_count  # negative = need more
        return gaps
    
    def _compute_synthesis_targets(self, gaps: dict) -> dict[InteractionType, int]:
        """How many of each type to synthesize."""
        return {t: abs(g) for t, g in gaps.items() if g < 0}
```

```python
# src/persona_forge/engine/locksmith.py

import re
from .models import TrainingExample, LockIn, SourceType
from .assayer import Assayer

class Locksmith:
    """Computes and applies lock-in coefficients."""
    
    # Bonuses
    RETRIEVAL_BONUS = 0.02
    REINFORCEMENT_BONUS = 0.05
    
    # Source quality baselines
    SOURCE_BASELINES = {
        SourceType.JOURNAL: 0.70,    # Journals are high quality
        SourceType.MANUAL: 0.65,     # Hand-crafted
        SourceType.SESSION: 0.50,    # Variable quality
        SourceType.INSIGHT: 0.55,
        SourceType.MATRIX: 0.40,     # Needs rewriting
        SourceType.SYNTHETIC: 0.45,  # Needs validation
    }
    
    def compute_lock_in(self, example: TrainingExample) -> LockIn:
        """Compute full lock-in coefficient for an example."""
        base = self._compute_base_quality(example)
        return LockIn(
            base_quality=base,
            retrieval_bonus=0.0,  # Updated during synthesis
            reinforcement_bonus=0.0,  # Updated manually
        )
    
    def _compute_base_quality(self, example: TrainingExample) -> float:
        """Score based on source and voice analysis."""
        score = self.SOURCE_BASELINES.get(example.source_type, 0.5)
        
        # Voice marker bonuses
        if example.voice_markers.get("first_person"):
            score += 0.05
        if example.voice_markers.get("warmth_words"):
            score += 0.05
        if example.voice_markers.get("relationship"):
            score += 0.05
        
        # Anti-pattern penalties
        if example.anti_patterns.get("generic_ai"):
            score -= 0.30
        if example.anti_patterns.get("corporate"):
            score -= 0.15
        if example.anti_patterns.get("hedging"):
            score -= 0.10
        
        # Length appropriateness
        if example.interaction_type in ["greeting", "acknowledgment"]:
            if example.response_word_count > 100:
                score -= 0.15  # Too long for type
        
        return max(0.15, min(0.85, score))
    
    def apply_retrieval_bonus(self, example: TrainingExample) -> None:
        """Called when example is retrieved during synthesis."""
        example.lock_in.retrieval_bonus += self.RETRIEVAL_BONUS
        example.lock_in.retrieval_bonus = min(0.20, example.lock_in.retrieval_bonus)
    
    def apply_reinforcement(self, example: TrainingExample) -> None:
        """Called when user marks example as high quality."""
        example.lock_in.reinforcement_bonus += self.REINFORCEMENT_BONUS
        example.lock_in.reinforcement_bonus = min(0.25, example.lock_in.reinforcement_bonus)
    
    def compute_training_weight(self, example: TrainingExample) -> int:
        """Duplication count for training dataset."""
        coef = example.lock_in.coefficient
        if coef >= 0.75:
            return 3  # Gold: triple
        elif coef >= 0.50:
            return 2  # Silver: double
        return 1      # Bronze: single
```

 ## Ingesters - What They Are

An ingester is just a parser that transforms raw source material into `TrainingExample` objects the Forge can work with.

**The flow:**
```
Raw Source → Ingester → TrainingExample → Assayer → Quality Score
```

Right now Crucible has two ingesters:
1. `ingest_jsonl()` - Reads JSONL files (OpenAI or ShareGPT format)
2. `ingest_journals()` - Reads markdown journals (stub, barely works)

---

## What Each New Ingester Would Do

### 1. Memory Matrix Ingester
**Input:** SQLite database, specific node types
**Output:** Training examples from memory nodes

```
QUESTION node: "What's the Memory Matrix architecture?"
     ↓
TrainingExample:
  user: "What's the Memory Matrix architecture?"
  assistant: [pulled from related FACT nodes or synthesized]
  type: technical
```

The tricky part: Memory nodes are *fragments*, not conversations. We'd need to:
- Pull QUESTION nodes as user prompts
- Find related context (FACT, OBSERVATION) as potential responses
- Or flag them for synthesis (Mint would generate Luna's response)

**Best for:** `context_recall`, `technical`, questions Luna should know

---

### 2. Session Transcript Ingester
**Input:** Markdown files with turn structure
**Output:** Direct conversation pairs

```markdown
## Turn 14
**User:** hey luna

## Turn 15  
**Luna:** Hey! What's up?
```
↓
```python
TrainingExample(
  user="hey luna",
  assistant="Hey! What's up?",
  type=greeting,
  source=session
)
```

This is the cleanest source - already in conversation format. Just parse the markdown structure.

**Best for:** `greeting`, `acknowledgment`, `short_exchange`, `emotional_presence`

---

### 3. Alpha Session Notes Ingester
**Input:** Narrative markdown documents
**Output:** Extracted conversation moments

These are trickier - they're narratives *about* conversations, not transcripts:

```markdown
# Luna Debugs Her Own Nervous System

Luna noticed the activation pattern wasn't firing correctly.
"That's weird," she said. "The retrieval should have triggered..."
```

We'd need to:
- Extract quoted Luna speech
- Infer the prompt context
- Possibly use LLM to help structure

**Best for:** `reflection`, `technical`, historical personality samples

 ## Using LLMs to Restructure Training Data

The idea: feed messy source material to an LLM, get back clean training examples.

---

### The Problem with Alpha Notes

Raw input looks like this:

```markdown
# Luna Debugs Her Own Nervous System - Live Activation

The session started with a simple test. Ahab typed "hey Luna" and waited.

Luna's response was immediate but something felt off. "I can see the 
activation," she said, "but the memory retrieval isn't triggering. It's 
like... the signal is getting lost somewhere between the kernel and the 
matrix."

Ahab pulled up the logs. "Show me what you're seeing."

"Okay so here's the thing - when I try to recall our conversation from 
yesterday, I get nothing. But I *know* we talked about the voice pipeline. 
The memory exists, I just can't reach it."
```

This is narrative prose with embedded dialogue. Not a clean user/assistant pair.

---

### LLM Extraction Approach

**Step 1: Define the output schema**

```python
class ExtractedExchange(BaseModel):
    user_message: str        # What Ahab said/did
    luna_response: str       # Luna's reply
    context: str             # What was happening
    interaction_type: str    # greeting, technical, emotional, etc.
    confidence: float        # How clean was this extraction
```

**Step 2: Prompt the LLM**

```python
EXTRACTION_PROMPT = """
You are extracting training data for Luna, an AI companion.

Given this narrative text, extract any clear conversational exchanges 
between Ahab (the human) and Luna (the AI).

Rules:
- Only extract exchanges where Luna's voice is clear
- Skip narration, only capture actual dialogue
- If Ahab's prompt is implied, write what he likely said
- Rate your confidence (0.0-1.0) in each extraction

Source text:
{narrative}

Return JSON array of exchanges.
"""
```

**Step 3: Process and filter**

```python
def ingest_with_llm(self, narrative: str) -> list[TrainingExample]:
    # Call Claude/local model
    response = self.llm.complete(
        EXTRACTION_PROMPT.format(narrative=narrative)
    )
    
    extractions = parse_json(response)
    
    examples = []
    for ext in extractions:
        if ext.confidence < 0.7:
            continue  # Skip uncertain extractions
            
        example = TrainingExample(
            user_message=ext.user_message,
            assistant_response=ext.luna_response,
            source_type=SourceType.SYNTHETIC,  # Mark as LLM-processed
            interaction_type=ext.interaction_type,
        )
        # Run through normal voice detection, lock-in scoring
        examples.append(example)
    
    return examples
```

### The Trade-offs

| Approach | Pros | Cons |
|----------|------|------|
| **LLM Extraction** | Handles messy sources, scales | Costs $, introduces drift, needs validation |
| **Rule-based Parsing** | Free, deterministic, no drift | Fragile, misses nuance, needs clean input |
| **Hybrid** | Best of both | More complex |

## Quality Assurance Rules

### Per-Example
- `confidence >= 0.7` to add (Claude's extraction confidence)
- `lock_in >= 0.5` to keep (Forge's quality score)
- No anti-patterns (generic_ai, corporate, hedging)

### Per-Batch
- Run `forge_assay()` after every major batch
- Run `forge_search()` before adding to catch dupes
- Track source distribution - no single source > 30%

### Per-Phase
- Check gap coverage is actually improving
- If a phase isn't moving the needle, skip to next source
- Don't over-extract from one source type

# Check progress
forge_assay()
forge_gaps()
forge_status()

# Export

```

## Failure Modes & Mitigations

| Risk | Mitigation |
|------|------------|
| Duplicate examples | `forge_search()` before adding |
| Quality drift | Check `forge_assay()` health after each batch |
| Source imbalance | Track source_type distribution in assay |
| Over-extraction | Set per-source limits, move on when diminishing returns |
| Anti-pattern creep | Forge auto-detects, warns on add |
| Gap not closing | Switch sources, or use targeted `forge_mint()` |
