## Three New Backend Pieces

### 1. Event Bus — the nervous system

Right now actors do their work silently. We need them to **announce** what they're doing.

```python
# src/luna/core/event_bus.py

class MemoryEvent:
    type: str          # "node_created", "edge_added", "access_recorded", etc.
    timestamp: float
    actor: str         # "scribe", "librarian", "matrix", "director"
    payload: dict      # type-specific data

class MemoryEventBus:
    """Pub/sub for memory operations. Actors publish, WebSocket relays."""
    
    _subscribers: list[Callable[[MemoryEvent], Awaitable[None]]]
    
    async def emit(self, event: MemoryEvent)
    def subscribe(self, callback) -> Callable  # returns unsubscribe
```

**Event types:**

| Event | Actor | Payload |
|---|---|---|
| `node_created` | Scribe | `{node_id, node_type, content_preview, confidence}` |
| `edge_added` | Librarian | `{from_id, to_id, relationship, strength}` |
| `edge_pruned` | Librarian | `{from_id, to_id, reason}` |
| `access_recorded` | Matrix | `{node_id, new_count, new_lock_in}` |
| `lock_in_changed` | Matrix | `{node_id, old_value, new_value, old_state, new_state}` |
| `cluster_updated` | Librarian | `{cluster_id, name, member_count, lock_in}` |
| `retrieval_started` | Director | `{query, budget_preset}` |
| `retrieval_completed` | Director | `{query, nodes_returned, tokens_used, clusters_hit}` |
| `spreading_activation` | Matrix/Graph | `{seed_nodes, activated: [{id, activation, hop}]}` |
| `constellation_assembled` | Director | `{clusters, nodes, total_tokens}` |
| `extraction_batch` | Scribe | `{session_id, objects: [...], edges: [...]}` |

**Integration cost:** Add `await self._event_bus.emit(...)` calls to ~8 places across Scribe, Librarian, Matrix actor, and Director. The bus itself is maybe 40 lines. Low cost, high leverage.

**Trade-offs made explicit:**
- **React app over artifact** — artifacts can't hold WebSocket connections or persist state. This needs to be a real app.
- **Separate app, not embedded in Eclissi** — keeps it independent, debuggable, throwaway-able. It's a prototyping tool, not a product.
- **Event bus over polling** — polling would work but adds latency and load. Events are the right primitive here.
- **Graph dump + events over pure events** — you need initial state. Events alone means missing everything before you connected.
- **D3/react-force-graph over the custom force layout in v2** — the v2 layout was cool but fragile. Real libraries handle pan/zoom/performance properly.

**What we lose:** Nothing. The v2 artifact design vocabulary (lock-in rings, cluster hulls, activation glow) all transfers directly.

## What Makes This a Good Sandbox

**Faithful to prod:**
- Same sqlite schema
- Same sqlite-vec for embeddings  
- Same lock-in formula (4 weights, same state machine)
- Same FTS5 + vector + RRF hybrid search
- Same spreading activation algorithm

**Better than prod for prototyping:**
- Every operation emits events (prod doesn't yet)
- `replay()` traces every phase (prod has no equivalent)
- Tuning params are hot-swappable (prod requires restart)
- Can reset/seed instantly
- No risk of corrupting real memories
- Can run pathological test data

**What it DOESN'T have:**
- No Scribe extraction pipeline (you add nodes manually or via seed)
- No Librarian filing logic (you add edges manually or via seed)  
- No Director inference (no LLM calls)
- No conversation context management
- No personality system

Those are out of scope. This is a **retrieval algorithm testbed**, not a full agent.

## Event Bus (the connective tissue)

```python
# observatory_mcp/event_bus.py

@dataclass
class MemoryEvent:
    type: str           # "node_created", "search_fts5", "activation_hop", etc.
    timestamp: float
    actor: str          # "matrix", "search", "activation", "cluster", "user"
    payload: dict

class EventBus:
    """In-memory pub/sub. WebSocket relay subscribes to this."""
    
    def __init__(self):
        self._subscribers: list[Callable] = []
        self._history: deque[MemoryEvent] = deque(maxlen=1000)
    
    async def emit(self, event: MemoryEvent):
        self._history.append(event)
        for sub in self._subscribers:
            await sub(event)
    
    def subscribe(self, callback) -> Callable:
        self._subscribers.append(callback)
        return lambda: self._subscribers.remove(callback)
    
    def recent(self, n=50, type_filter=None) -> list[MemoryEvent]:
        ...
```

**Event types for retrieval replay** (the juicy ones):

```
search_started      → {query, method, params}
search_fts5_done    → {results: [{id, score, snippet}], ms}
search_vector_done  → {results: [{id, score, similarity}], ms}
search_fusion_done  → {results: [{id, rrf_score, fts5_rank, vec_rank}], ms}
activation_started  → {seed_ids, params}
activation_hop      → {hop: 1, activated: [{id, activation, source_id, edge_type}]}
activation_done     → {total_activated, by_hop: {0: n, 1: n, 2: n}}
cluster_activated   → {clusters: [{id, name, activation, member_overlap}]}
constellation_built → {included: [...], excluded: [...], tokens_used, budget}
```

Frontend subscribes to WebSocket, renders events as they arrive. You type a query in Claude → `sandbox_recall` fires → events stream to the Observatory → you watch the retrieval unfold node by node.
