## The Smells

### Smell #1: iTerm2 + Browser is Weird

The spec describes launching iTerm2 for TUI, *and* a separate browser window, *and* using Rectangle/AppleScript to position them side-by-side.

**Why not just... a web app?**

A single React app with:
- Timeline scrubber (left)
- Orb preview (right)
- Parameter sliders (bottom)

You lose nothing, gain:
- Cross-platform (works on Mars College whatever-hardware)
- No AppleScript fragility
- Hot reload while tweaking
- Shareable (screenshare, remote pairing)

The TUI feels like complexity for vibes. Unless there's a specific reason it needs to be terminal-based (there isn't one stated).

### Smell #2: Two WebSocket Servers?

```
Port 3000 → Vite preview server
Port 3001 → Performance Studio state sync server
Port 8000 → Luna Hub API (existing)
```

This is three things that need to be running, coordinated, and kept in sync. 

**Simpler:** Just use Luna Hub's existing WebSocket at 8000, add a `/ws/performance` endpoint (which you've already specced), and have the Performance Studio UI be a single-page app that connects to that one socket.

### Smell #3: Chicken-and-Egg with Timestamps

The `phrase_performance` table has `start_time` and `end_time`. But where do these come from?

1. If they come from TTS audio → you need to synthesize audio first
2. But synthesis needs prosody parameters → which you're trying to tune

You can't scrub a timeline of audio you haven't generated yet. The spec glosses over this. In practice, you'd need to:
- Generate audio once with baseline prosody
- Scrub *that* audio
- Tune prosody
- Regenerate audio to hear changes

That's a more complex workflow than depicted.

### Smell #4: MCP Tool for MVP?

The MCP integration (`luna_tune_performance`) adds significant complexity for a training tool. You need:
- WebSocket connection management
- Bidirectional state sync
- Error handling for disconnected agents

For an MVP, just make a standalone web UI. Add MCP later when you actually want agents collaborating on tuning. This feels like scope creep.
