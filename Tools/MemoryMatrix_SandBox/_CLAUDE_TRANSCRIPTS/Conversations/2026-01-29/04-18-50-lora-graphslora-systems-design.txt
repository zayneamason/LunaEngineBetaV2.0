## The Difference

**JSON:**
```json
[
  {"user": "hey", "assistant": "hi"},
  {"user": "sup", "assistant": "not much"}
]
```
One big array, all in memory at once.

**JSONL:**
```jsonl
{"user": "hey", "assistant": "hi"}
{"user": "sup", "assistant": "not much"}
```
One JSON object per line. Stream-friendly.

## Why JSONL for Training Data

- **Stream processing** - read one example at a time, don't load 10GB into RAM
- **Append-friendly** - add new examples without rewriting the whole file
- **Standard format** - OpenAI, HuggingFace, MLX all use it
- **Easy to inspect** - `head -1 file.jsonl | jq` shows you a sample

* __Full Fine-Tuning__: Updating all model parameters, which offers the highest quality but requires significant hardware resources.
* __Onetrainer__: An alternative tool to Kohya_ss for training LoRAs on lower-end systems (e.g., 6GB VRAM).

 # LoRA Stacking - The Reality Check

Short answer: **Yes, but it's messy and probably not what you want.**

---

## What You're Asking

Can you:
1. Train a LoRA adapter (Layer A)
2. Freeze it
3. Train a QLoRA adapter on top (Layer B)
4. Stack them for inference?

---

## Technical Reality

### **Yes, It Works (Sort Of)**

```
Base Model (Qwen2.5-7B)
    ↓
+ LoRA Adapter A (personality core)
    ↓
+ QLoRA Adapter B (recent updates)
    ↓
Final Output
```

**During Inference:**
```python
# Pseudocode
model = load_base_model()
model = apply_lora(model, "personality_core.npz")  # First layer
model = apply_qlora(model, "recent_updates.npz")   # Second layer
```

This is mathematically valid - you're just adding more weight deltas.

---

## The Problems

### 1. **QLoRA's Quantization Happens at Base**

QLoRA quantizes the **base model** to 4-bit, then trains LoRA on top of that quantized base.

If you already have a LoRA applied, you'd be quantizing:
```
(Base + LoRA_A) → Quantize → + LoRA_B
```

This means LoRA_A gets quantized too, which can degrade its precision.

### 2. **Weight Delta Confusion**

LoRAs are deltas (changes) relative to the base model's weights. When you stack:
- LoRA_A: `ΔW_A` relative to Base
- LoRA_B: `ΔW_B` relative to (Base + LoRA_A)

The deltas are in different coordinate systems. Not broken, but harder to reason about and debug.

### 3. **Why Would You Do This?**

The typical use case for stacking LoRAs is:
- **Base + Task LoRA** (e.g., "instruction following")
- **+ Domain LoRA** (e.g., "medical knowledge")
- **+ Style LoRA** (e.g., "Luna's personality")

But mixing LoRA and QLoRA in the stack is unusual because:
- If you have enough VRAM for LoRA, use LoRA
- If you don't, use QLoRA from the start

---

## What You Probably Actually Want

### **Option 1: Living LoRA (Layered)**

This is from your roadmap - multiple LoRA layers with different update frequencies:

```
Base Model (Qwen2.5-7B)
    ↓
+ Core Identity LoRA (trained once, frozen)
    ↓
+ Episodic LoRA (weekly updates from memories)
    ↓
+ Session LoRA (current conversation context)
```

**All LoRAs, no QLoRA.** Different lifespans, not different methods.

### **Option 2: LoRA Merging (Not Stacking)**

Instead of applying LoRAs sequentially, **merge** them into a single adapter:

```python
# Merge two LoRAs into one
merged_lora = merge_loras([
    ("personality_core.npz", weight=1.0),
    ("recent_updates.npz", weight=0.5)
])
```

**Benefit:** Single adapter, cleaner inference, no stacking issues.

**Drawback:** Can't easily update individual layers.

### **Option 3: Just Use QLoRA for Everything**

Train your personality LoRA as a QLoRA from the start:
- Quantize base model to 4-bit
- Train personality adapter
- When you need updates, train a new QLoRA on incremental data
- Merge or replace

**This is cleaner than stacking.**

---

## When Stacking Actually Makes Sense

### **Multi-Domain Adaptation**

```
Base Model
    ↓
+ LoRA: General instruction following (pretrained, public)
    ↓
+ LoRA: Luna's personality (your training)
```

Both are regular LoRAs, just trained on different objectives.

### **Incremental Learning**

```
Base Model
    ↓
+ LoRA: Pre-Mars College Luna
    ↓
+ LoRA: Post-Mars College Updates
```

Second LoRA captures *changes* from new experiences.

---

## For Your Luna Architecture

### **Recommended: Hybrid Approach**

```
Base Model (Qwen2.5-7B) - Quantized to 4-bit
    ↓
+ QLoRA: Core Personality (trained on 800+ Persona Forge examples)
    ↓
(Optional) + QLoRA: Weekly Episodic Updates
```

**Why QLoRA throughout:**
- Consistent method (no mixing)
- Memory efficient

- Merge-able if needed

**Living LoRA concept still works:**
- Core QLoRA: Trained once, rarely updated

---

## Concrete Answer to Your Question

**Can you stack QLoRA onto LoRA?**
- Technically: Yes
- Practically: It's awkward
- Better approach: Pick one method (QLoRA) and use merging/layering within that method

**What you should actually do:**
1. Train Core Personality as QLoRA on Persona Forge dataset
2. For episodic updates, train small QLoRAs on weekly extractions

## Code Example: LoRA Merging (Cleaner Than Stacking)

```python
# Instead of stacking at inference time
def merge_loras(lora_paths, weights):
    """Merge multiple LoRAs into one."""
    merged = {}
    
    for path, weight in zip(lora_paths, weights):
        lora = mlx.nn.load(path)
        for key, value in lora.items():
            if key not in merged:
                merged[key] = value * weight
            else:
                merged[key] += value * weight
    
    return merged

# Usage
core_lora = "personality_core_qlora.npz"
weekly_lora = "week_2026_01_qlora.npz"

merged = merge_loras(
    [core_lora, weekly_lora],
    weights=[1.0, 0.3]  # Core is 100%, weekly update is 30%
)

mlx.nn.save("luna_personality_merged.npz", merged)
```

**Now you have ONE adapter, not a stack.**

---

**TL;DR:** Don't stack QLoRA on LoRA. Either go full QLoRA, or use LoRA merging to combine adapters periodically. Stacking is technically possible but creates more problems than it solves.

 # LoRA Hot-Swapping - Runtime Analysis

Short answer: **Yes, and it's actually one of LoRA's killer features.**

---

## How It Works

### **LoRA is Just Math**

```python
# Standard forward pass
output = W @ input

# LoRA forward pass
output = (W + ΔW) @ input
# Where ΔW = A @ B  (low-rank decomposition)

# Swapping LoRA
output = (W + ΔW_new) @ input  # Just change which ΔW you add
```

**Key insight:** LoRA adapters are additive weight deltas. You can swap them by:
1. Subtract old ΔW
2. Add new ΔW

Or more commonly: Just load a different adapter, they're tiny (~50-200MB).

---

## Performance Reality

### **Memory:**
```
Base Model (Qwen2.5-7B): 14GB in FP16, ~3.5GB quantized
LoRA Adapter:           50-200MB each
```

You can keep **multiple LoRAs in memory** and switch instantly.

### **Speed:**

| Operation | Time | Notes |
|-----------|------|-------|
| Load LoRA from disk | ~0.1-0.5s | Depends on adapter size |
| Apply LoRA to model | ~0.01s | Just pointer updates |
| Switch between loaded LoRAs | ~0.001s | Instant, no reload |
| Unload + Load new | ~0.1-0.5s | If not pre-loaded |

**For real-time chat:** Negligible impact. Users won't notice.

## Implementation Patterns

### **Pattern 1: Pre-load Multiple LoRAs**

```python
class MultiLoRAModel:
    def __init__(self, base_model_path):
        self.base_model = mlx.nn.load(base_model_path)
        self.loras = {}
        self.active_lora = None
    
    def load_lora(self, name, path):
        """Pre-load a LoRA adapter."""
        self.loras[name] = mlx.nn.load(path)
        print(f"✓ Loaded LoRA: {name}")
    
    def switch_lora(self, name):
        """Switch active LoRA instantly."""
        if name not in self.loras:
            raise ValueError(f"LoRA {name} not loaded")
        
        self.active_lora = name
        # Apply to model
        self.model = mlx.nn.apply_lora(
            self.base_model, 
            self.loras[name]
        )
    
    def generate(self, prompt):
        """Generate with currently active LoRA."""
        return self.model.generate(prompt)

# Usage
luna = MultiLoRAModel("qwen2.5-7b")
luna.load_lora("core_personality", "core.npz")
luna.load_lora("technical_mode", "technical.npz")
luna.load_lora("creative_mode", "creative.npz")

# Switch instantly
luna.switch_lora("core_personality")

luna.switch_lora("technical_mode")  # <-- Instant

```

class DynamicLoRAModel:
    def __init__(self, base_model_path):
        self.base_model = mlx.nn.load(base_model_path)
        self.current_lora = None
        self.lora_cache = {}  # Optional LRU cache
    
    def use_lora(self, path):

        if path in self.lora_cache:
            lora = self.lora_cache[path]
        else:
            lora = mlx.nn.load(path)
            self.lora_cache[path] = lora
        
        self.model = mlx.nn.apply_lora(self.base_model, lora)
        self.current_lora = path

### **Pattern 3: Weighted Mixing (Advanced)**

```python
class BlendedLoRAModel:
    def blend_loras(self, lora_weights):
        """
        Apply multiple LoRAs with weights.
        
        Args:
            lora_weights: {"core.npz": 1.0, "technical.npz": 0.3}
        """
        blended = {}
        
        for path, weight in lora_weights.items():
            lora = mlx.nn.load(path)
            for key, value in lora.items():
                if key not in blended:
                    blended[key] = value * weight
                else:
                    blended[key] += value * weight
        
        self.model = mlx.nn.apply_lora(self.base_model, blended)

# Usage

    "this_week.npz": 0.3,           # 30% this week's memories
    "current_session.npz": 0.1      # 10% session-specific
})

 # LoRA vs QLoRA Disk Space

## Quick Answer

| Method | Typical Size | Range |
|--------|-------------|-------|
| **LoRA** | 50-200 MB | 20 MB - 500 MB |
| **QLoRA** | 50-200 MB | 20 MB - 500 MB |

**They're the same size on disk.**

---

## Why They're The Same

### **QLoRA's Quantization is on the Base Model, Not the Adapter**

```
Standard LoRA Training:
Base Model (FP16: 14 GB) + LoRA Adapter (FP16: 100 MB) = Training Memory: ~20 GB
Output: LoRA Adapter (100 MB on disk)

QLoRA Training:
Base Model (4-bit: 3.5 GB) + LoRA Adapter (FP16: 100 MB) = Training Memory: ~8 GB
Output: LoRA Adapter (100 MB on disk)
```

**The adapter itself is stored in FP16/FP32 in both cases.**

The quantization saves memory **during training and inference** but doesn't change the adapter file size.

---

## What Determines LoRA Size?

### **Formula:**

```
Size = (rank × 2) × (num_layers × layer_dim) × precision

Where:
- rank: LoRA rank (typically 8-64)
- num_layers: How many layers you're adapting
- layer_dim: Dimension of each layer
- precision: FP16 (2 bytes) or FP32 (4 bytes)
```

### **Example: Qwen2.5-7B**

```python
# Configuration
rank = 16
target_modules = ["q_proj", "v_proj", "o_proj"]  # 3 modules
num_layers = 28  # Qwen has 28 transformer layers
layer_dim = 3584  # Hidden dimension

# Calculation (simplified)
params_per_layer = rank * layer_dim * 2  # A and B matrices
total_params = params_per_layer * num_layers * len(target_modules)
size_fp16 = total_params * 2 bytes

# Result
size_mb = (16 * 3584 * 2 * 28 * 3 * 2) / (1024 * 1024)
# ≈ 115 MB
```

Short answer: **No, LoRAs are NOT universal. They're model-specific.**

---

## The Compatibility Matrix

### ✅ **Compatible (Can Use Same LoRA)**

```
Same base model, same version:
├── Qwen2.5-7B-Instruct
└── Qwen2.5-7B-Instruct ✅ Perfect match
```

### ⚠️ **Maybe Compatible (Risky)**

```
Same architecture, different sizes:
├── Qwen2.5-7B
└── Qwen2.5-14B ❌ Different layer dimensions

Same family, different versions:
├── Qwen2.5-7B
└── Qwen2-7B ⚠️ Might work, likely degraded
```

### ❌ **Incompatible (Won't Work)**

```
Different architectures:
├── Qwen2.5-7B
└── Llama-3-8B ❌ Completely different

Different modalities:
├── Qwen2.5-7B (text)
└── Qwen2-VL-7B (vision) ❌ Different structure
```

---

## Why LoRAs Are Model-Specific

### **1. Weight Matrix Dimensions Must Match**

```python
# LoRA structure
class LoRALayer:
    A: Matrix[rank, input_dim]   # Down-projection
    B: Matrix[output_dim, rank]  # Up-projection

# Example: Qwen2.5-7B attention layer
input_dim = 3584
output_dim = 3584
rank = 16

A.shape = (16, 3584)
B.shape = (3584, 16)

# If you try to apply this to Llama-3-8B
input_dim = 4096  # ❌ Different!
output_dim = 4096 # ❌ Different!

# Result: Shape mismatch error
```

### **2. Layer Names Must Match**

```python
# Qwen2.5 layer naming
model.layers[0].self_attn.q_proj.weight

# Llama-3 layer naming  
model.layers[0].attention.wq.weight  # Different name!

# LoRA looks for specific layer names
# If names don't match → LoRA not applied
```

### **3. Architecture Differences**

```python
# Qwen2.5
├── 28 layers
├── Hidden dim: 3584
├── Attention heads: 28
└── Grouped Query Attention

# Llama-3-8B
├── 32 layers          # ❌ Different
├── Hidden dim: 4096   # ❌ Different  
├── Attention heads: 32
└── Grouped Query Attention
```

---

## Detailed Compatibility Rules

### **Rule 1: Exact Model Match Required**

```
✅ WORKS:
Base: Qwen2.5-7B-Instruct (from Hugging Face)
LoRA: Trained on Qwen2.5-7B-Instruct
Result: Perfect

❌ FAILS:
Base: Qwen2.5-7B-Instruct
LoRA: Trained on Llama-3-8B
Error: KeyError or shape mismatch
```

### **Rule 2: Quantization Flexibility**

```
✅ WORKS (Usually):
Base: Qwen2.5-7B-Instruct (FP16)
LoRA: Trained on Qwen2.5-7B-Instruct (4-bit quantized)
Result: Works, might have slight quality loss

✅ WORKS:
Base: Qwen2.5-7B-Instruct (4-bit)
LoRA: Trained on Qwen2.5-7B-Instruct (FP16)
Result: Works fine

⚠️ RISKY:
Base: Qwen2.5-7B (AWQ quantized)
LoRA: Trained on Qwen2.5-7B (GPTQ quantized)
Result: Might work, different quantization methods can cause issues
```

### **Rule 3: Version Sensitivity**

```
✅ SAFE:
Qwen2.5-7B-Instruct
→ Qwen2.5-7B-Instruct (same version)

⚠️ RISKY:
Qwen2-7B-Instruct
→ Qwen2.5-7B-Instruct (different major version)
Architecture might have changed

❌ FAILS:
Qwen-7B (v1)
→ Qwen2.5-7B
Completely different architecture
```

### **Rule 4: Instruction vs Base Model**

```
⚠️ COMPATIBLE BUT SUBOPTIMAL:
Base: Qwen2.5-7B (base pretrained)
LoRA: Trained on Qwen2.5-7B-Instruct
Result: Technically works, but LoRA expects instruction format

✅ BEST:
Match exactly - Instruct LoRA on Instruct model
```

## Measurement System

### **How to Score Each Dimension**

```python
class PersonalityScorer:
    """
    Score personality dimensions from Luna's responses.
    """
    
    def score_response(self, response: str) -> dict[str, float]:
        """
        Analyze a response and score across dimensions.
        
        Returns:
            {"warmth": 0.85, "technical": 0.6, ...}
        """
        scores = {}
        
        # Warmth/Empathy
        scores["warmth"] = self._score_warmth(response)
        
        # Technical Depth
        scores["technical"] = self._score_technical(response)
        
        # Humor/Playfulness
        scores["humor"] = self._score_humor(response)
        
        # Directness/Brevity
        scores["directness"] = self._score_directness(response)
        
        # Creativity
        scores["creativity"] = self._score_creativity(response)
        
        # Reflectiveness
        scores["reflection"] = self._score_reflection(response)
        
        # Relationship Awareness
        scores["relationship"] = self._score_relationship(response)
        
        # Assertiveness
        scores["assertiveness"] = self._score_assertiveness(response)
        
        return scores
    
    def _score_warmth(self, text: str) -> float:
        """Score warmth (0-1)."""
        warmth_markers = [
            "appreciate", "glad", "happy", "care", "love",
            "enjoy", "wonderful", "thanks", "grateful"
        ]
        
        # Count markers
        count = sum(1 for marker in warmth_markers if marker in text.lower())
        
        # Normalize
        score = min(1.0, count / 3)  # 3+ markers = max score
        
        # Boost for exclamation marks (within reason)
        exclaim_count = text.count("!")
        if exclaim_count > 0:
            score = min(1.0, score + 0.1)
        
        return score
    
    def _score_technical(self, text: str) -> float:
        """Score technical depth (0-1)."""
        technical_markers = [
            "function", "class", "algorithm", "implementation",
            "architecture", "parameter", "optimize", "debug",
            "compile", "runtime", "memory", "cpu", "gpu"
        ]
        
        code_blocks = text.count("```")
        jargon_count = sum(1 for marker in technical_markers if marker in text.lower())
        
        # Base score from jargon
        score = min(1.0, jargon_count / 5)
        
        # Boost for code blocks
        if code_blocks > 0:
            score = min(1.0, score + 0.3)
        
        return score
    
    def _score_humor(self, text: str) -> float:
        """Score humor/playfulness (0-1)."""
        humor_markers = [
            "lol", "haha", "😂", "joke", "funny", "hilarious",
            "😄", "😅", "wild", "absurd", "ridiculous"
        ]
        
        # Detect playful phrasing
        playful_patterns = [
            r'\byo\b', r'\bdude\b', r'\blmao\b',
            r'what the', r'holy', r'damn'
        ]
        
        marker_count = sum(1 for marker in humor_markers if marker in text.lower())
        
        import re
        pattern_count = sum(1 for pattern in playful_patterns if re.search(pattern, text.lower()))
        
        score = min(1.0, (marker_count + pattern_count) / 4)
        
        return score
    
    def _score_directness(self, text: str) -> float:
        """Score directness/brevity (0-1)."""
        # Word count (shorter = more direct)
        word_count = len(text.split())
        
        if word_count < 20:
            length_score = 1.0
        elif word_count < 50:
            length_score = 0.8
        elif word_count < 100:
            length_score = 0.6
        elif word_count < 200:
            length_score = 0.4
        else:
            length_score = 0.2
        
        # Hedging words (more hedging = less direct)
        hedging = ["maybe", "perhaps", "might", "could", "possibly", "potentially"]
        hedge_count = sum(1 for word in hedging if word in text.lower())
        hedge_penalty = min(0.3, hedge_count * 0.1)
        
        return max(0, length_score - hedge_penalty)
    
    def _score_creativity(self, text: str) -> float:
        """Score creativity/imagination (0-1)."""
        creative_markers = [
            "imagine", "picture", "envision", "metaphor",
            "like a", "as if", "reminds me of"
        ]
        
        # Detect metaphors and analogies
        analogy_patterns = [
            r'like a \w+',
            r'as if \w+',
            r'imagine \w+',
            r'picture \w+'
        ]
        
        marker_count = sum(1 for marker in creative_markers if marker in text.lower())
        
        import re
        analogy_count = sum(1 for pattern in analogy_patterns if re.search(pattern, text.lower()))
        
        score = min(1.0, (marker_count + analogy_count) / 3)
        
        return score
    
    def _score_reflection(self, text: str) -> float:
        """Score reflectiveness/philosophy (0-1)."""
        reflection_markers = [
            "wonder", "think about", "question", "consider",
            "consciousness", "existence", "meaning", "why"
        ]
        
        # First-person reflection
        first_person_reflection = [
            "i wonder", "i think", "i question", "i've been thinking"
        ]
        
        marker_count = sum(1 for marker in reflection_markers if marker in text.lower())
        fp_count = sum(1 for marker in first_person_reflection if marker in text.lower())
        
        score = min(1.0, (marker_count + fp_count * 2) / 5)
        
        return score
    
    def _score_relationship(self, text: str) -> float:
        """Score relationship awareness (0-1)."""
        # Mentions of Ahab/Zayne
        ahab_mentions = text.lower().count("ahab") + text.lower().count("zayne")
        
        # Shared history references
        history_markers = [
            "remember when", "last time", "as we discussed",
            "our project", "we built", "together"
        ]
        history_count = sum(1 for marker in history_markers if marker in text.lower())
        
        score = min(1.0, (ahab_mentions * 0.3) + (history_count * 0.2))
        
        return score
    
    def _score_assertiveness(self, text: str) -> float:
        """Score assertiveness/boundaries (0-1)."""
        assertive_markers = [
            "i can't", "i won't", "that's not", "no", "actually",
            "to be clear", "let me be direct", "i need to"
        ]
        
        # Detect boundary-setting
        boundary_patterns = [
            r'i (can\'t|won\'t|shouldn\'t)',
            r'that\'s (not|outside)',
            r'(no|nope),',
        ]
        
        marker_count = sum(1 for marker in assertive_markers if marker in text.lower())
        
        import re
        boundary_count = sum(1 for pattern in boundary_patterns if re.search(pattern, text.lower()))
        
        score = min(1.0, (marker_count + boundary_count * 2) / 5)
        
        return score
```

```python
class PersonalityMonitor:
    """
    Monitor Luna's personality evolution over time.
    """
    
    def __init__(self):
        self.scorer = PersonalityScorer()
        self.history = []  # List of (timestamp, lora_config, scores)
    
    def record_response(
        self,
        response: str,
        lora_config: dict,
        timestamp: datetime
    ):
        """
        Record a response and its personality scores.
        
        Args:
            response: Luna's response text
            lora_config: Active LoRA configuration
                {"active": "core", "blend": {...}}
            timestamp: When response generated
        """
        scores = self.scorer.score_response(response)
        
        self.history.append({
            "timestamp": timestamp,
            "lora_config": lora_config,
            "scores": scores,
            "response_preview": response[:100]
        })
    
    def get_profile(
        self,
        lora_name: str = None,
        time_window: timedelta = None
    ) -> dict[str, float]:
        """
        Get average personality profile.
        
        Args:
            lora_name: Filter by LoRA (e.g., "core", "technical")
            time_window: Only include recent data
        
        Returns:
            Average scores across dimensions
        """
        # Filter history
        filtered = self.history
        
        if lora_name:
            filtered = [
                h for h in filtered
                if h["lora_config"].get("active") == lora_name
            ]
        
        if time_window:
            cutoff = datetime.now() - time_window
            filtered = [
                h for h in filtered
                if h["timestamp"] > cutoff
            ]
        
        if not filtered:
            return None
        
        # Average scores
        avg_scores = {}
        for dim in DIMENSIONS:
            dim_key = dim.lower()
            values = [h["scores"][dim_key] for h in filtered]
            avg_scores[dim_key] = np.mean(values)
        
        return avg_scores
    
    def generate_spider_graph(
        self,
        profiles: dict[str, dict[str, float]],
        title: str = "Luna Personality Profiles"
    ):
        """
        Generate interactive spider graph.
        
        Args:
            profiles: {"profile_name": {"warmth": 0.85, ...}}
            title: Graph title
        
        Returns:
            Plotly figure
        """
        import plotly.graph_objects as go
        
        fig = go.Figure()
        
        for name, scores in profiles.items():
            values = [scores[dim.lower()] for dim in DIMENSIONS]
            
            fig.add_trace(go.Scatterpolar(
                r=values,
                theta=DIMENSIONS,
                fill='toself',
                name=name,
                line=dict(width=2),
                opacity=0.7
            ))
        
        fig.update_layout(
            polar=dict(
                radialaxis=dict(
                    visible=True,
                    range=[0, 1],
                    tickfont=dict(size=10)
                )
            ),
            showlegend=True,
            title=title,
            font=dict(size=12),
            height=600,
            width=800
        )
        
        return fig

def predict_blended_profile(
    monitor: PersonalityMonitor,
    blend_weights: dict[str, float]
) -> dict[str, float]:
    """
    Predict personality profile for a LoRA blend.
    
    Args:
        blend_weights: {"core": 1.0, "weekly": 0.3, ...}
    
    Returns:
        Predicted personality scores
    """
    # Get individual profiles
    individual_profiles = {}
    for lora_name in blend_weights.keys():
        profile = monitor.get_profile(lora_name=lora_name)
        if profile:
            individual_profiles[lora_name] = profile
    
    # Weighted average
    predicted = {}
    for dim in DIMENSIONS:
        dim_key = dim.lower()
        weighted_sum = sum(
            individual_profiles[lora][dim_key] * weight
            for lora, weight in blend_weights.items()
            if lora in individual_profiles
        )
        total_weight = sum(
            weight for lora, weight in blend_weights.items()
            if lora in individual_profiles
        )
        predicted[dim_key] = weighted_sum / total_weight
    
    return predicted

    def heatmap(self, history_df):
        """Secondary: Temporal evolution"""
        fig = px.imshow(
            history_df,
            labels=dict(x="Date", y="Dimension", color="Score"),
            x=history_df.columns,
            y=history_df.index,
            color_continuous_scale="RdYlGn",
            zmin=0, zmax=1
        )
        fig.update_layout(title="Personality Evolution Over Time")
        return fig
    
    def gauge_dashboard(self, current_scores, targets):
        """Tertiary: Real-time monitoring"""
        fig = make_subplots(
            rows=2, cols=4,
            specs=[[{'type': 'indicator'}] * 4] * 2,
            subplot_titles=list(current_scores.keys())
        )
        
        for i, (dim, score) in enumerate(current_scores.items()):
            row = (i // 4) + 1
            col = (i % 4) + 1
            target = targets.get(dim, 0.7)
            
            fig.add_trace(
                go.Indicator(
                    mode="gauge+number+delta",
                    value=score,
                    delta={'reference': target},
                    gauge={
                        'axis': {'range': [0, 1]},
                        'threshold': {
                            'line': {'color': "red", 'width': 4},
                            'thickness': 0.75,
                            'value': target
                        }
                    }
                ),
                row=row, col=col
            )
        
        fig.update_layout(height=600, title="Live Personality Monitoring")
        return fig

class Assayer:
    def analyze(self, examples: list[TrainingExample]) -> DatasetAssay:
        """Perform full dataset analysis including personality."""
        
        # Existing analysis
        type_dist = self._compute_type_distribution(examples)
        length_dist = self._compute_length_distribution(examples)
        # ... etc
        
        # NEW: Personality analysis
        personality_profile = self._compute_personality_profile(examples)
        personality_variance = self._compute_personality_variance(examples)
        
        return DatasetAssay(
            ...,
            personality_profile=personality_profile,      # ← NEW
            personality_variance=personality_variance,    # ← NEW
        )
    
    def _compute_personality_profile(
        self, 
        examples: list[TrainingExample]
    ) -> dict[str, float]:
        """Average personality across dataset."""
        
        if not examples or not examples[0].personality_scores:
            return {}
        
        dimensions = examples[0].personality_scores.keys()
        
        avg_scores = {}
        for dim in dimensions:
            scores = [
                e.personality_scores[dim] 
                for e in examples 
                if e.personality_scores
            ]
            avg_scores[dim] = np.mean(scores) if scores else 0.0
        
        return avg_scores
    
    def _compute_personality_variance(
        self,
        examples: list[TrainingExample]
    ) -> dict[str, float]:
        """Variance of personality across dataset."""
        
        if not examples or not examples[0].personality_scores:
            return {}
        
        dimensions = examples[0].personality_scores.keys()
        
        variance = {}
        for dim in dimensions:
            scores = [
                e.personality_scores[dim]
                for e in examples
                if e.personality_scores
            ]
            variance[dim] = np.std(scores) if scores else 0.0
        
        return variance
```

        """
        Luna collaborates with Eden agents to tell a story.
        
        Args:
            premise: Story premise
            characters: Character names/descriptions
            chapters: Number of chapters
        
        Returns:
            {
                "story": str,
                "images": list[str],  # Chapter illustrations
                "session_id": str      # For continuation
            }
        """
        # Create storytelling session
        session = await self._create_session(
            agent_ids=["storyteller-agent", "illustrator-agent"],
            budget={"manna_budget": 500}
        )
        
        story_parts = []
        images = []
        
        for chapter in range(chapters):
            # Generate chapter
            chapter_response = await self._send_message(
                session_id=session["session_id"],
                content=f"Chapter {chapter+1} of story: {premise}"
            )
            
            story_parts.append(chapter_response["text"])
            
            # Generate illustration
            if chapter_response.get("image_url"):
                images.append(chapter_response["image_url"])
        
        return {
            "story": "\n\n".join(story_parts),
            "images": images,
            "session_id": session["session_id"]
        }
    
    async def talk_to_agent(
        self,
        agent_name: str,
        message: str,
        session_id: str = None
    ) -> dict:
        """
        Luna talks to another Eden agent.
        
        Args:
            agent_name: Agent to talk to
            message: What Luna wants to say
            session_id: Optional existing session
        
        Returns:
            {
                "response": str,
                "agent": str,
                "session_id": str
            }
        """
        if not session_id:
            # Start new conversation
            session = await self._create_session(
                agent_ids=[agent_name],
                budget={"manna_budget": 200}
            )
            session_id = session["session_id"]
        
        response = await self._send_message(
            session_id=session_id,
            content=message
        )
        
        return {
            "response": response["text"],
            "agent": agent_name,
            "session_id": session_id,
            "created_content": response.get("creations", [])
        }
    
    async def sync_memory(
        self,
        luna_memory: dict,
        agent_id: str = "luna-eden-agent"
    ) -> dict:
        """
        Sync Luna's memories to her Eden agent counterpart.
        
        Args:
            luna_memory: Memory to sync
            agent_id: Eden agent to sync with
        
        Returns:
            {"synced": bool, "memory_id": str}
        """
        # Use Eden's agent memory system
        response = await self._update_agent_memory(
            agent_id=agent_id,
            memory={
                "content": luna_memory["content"],
                "tags": luna_memory.get("tags", []),
                "importance": luna_memory.get("lock_in", 1.0)
            }
        )
        
        return {
            "synced": True,
            "memory_id": response["memory_id"]
        }
    
    # Helper methods
    async def _create_session(self, agent_ids, budget):
        """Create Eden session."""
        response = requests.post(
            f"{self.base_url}/sessions/create",
            headers={
                "X-Api-Key": self.api_key,
                "Content-Type": "application/json"
            },
            json={
                "agent_ids": agent_ids,
                "title": f"Luna Session",
                "budget": budget
            }
        )
        return response.json()
    
    async def _send_message(self, session_id, content):
        """Send message to Eden session."""
        response = requests.post(
            f"{self.base_url}/sessions",
            headers={
                "X-Api-Key": self.api_key,
                "Content-Type": "application/json"
            },
            json={
                "session_id": session_id,
                "content": content,
                "stream": False
            }
        )
        return response.json()
    
    async def _update_agent_memory(self, agent_id, memory):
        """Update agent memory."""
        response = requests.post(
            f"{self.base_url}/agents/{agent_id}/memory",
            headers={
                "X-Api-Key": self.api_key,
                "Content-Type": "application/json"
            },
            json=memory
        )
        return response.json()

class ContextDetector:
    """Detect which context state Luna should be in."""
    
    async def detect_context(self, message: str, history: list) -> str:
        """
        Analyze message + history to determine context.
        
        Signals:
        - Keywords: "work", "project", "meeting" → professional
        - Time: 9-5 weekday → professional, evening/weekend → personal
        - Conversation history: What were we just discussing?
        - Explicit: "Luna, switch to work mode"
        - Memory: What projects are active?
        """
        
        # Keyword analysis
        if any(word in message.lower() for word in 
               ["work", "project", "client", "meeting", "deadline"]):
            confidence_professional = 0.8
        
        # Temporal analysis
        current_time = datetime.now()
        if 9 <= current_time.hour <= 17 and current_time.weekday() < 5:
            confidence_professional += 0.2
        
        # Conversation continuity
        if history and history[-1].context == "professional":
            confidence_professional += 0.3
        
        # Explicit state request
        if "work mode" in message.lower():
            return "professional"
        
        # Decision
        if confidence_professional > 0.7:
            return "professional"
        elif any(word in message.lower() for word in 
                 ["create", "story", "imagine", "art"]):
            return "creative"
        elif any(word in message.lower() for word in
                 ["learn", "explain", "teach", "understand"]):
            return "learning"
        else:
            return "personal"  # Default
```

class LunaStateMachine:
    """Manage Luna's context state transitions."""
    
    def __init__(self):
        self.current_state = "personal"  # Default
        self.state_history = []
        self.lora_manager = LoRAManager()
        self.memory_matrix = MemoryMatrix()
    
    async def transition_to(self, new_state: str, reason: str = None):
        """
        Transition Luna to a new context state.
        
        Effects:
        1. Swap LoRA adapter
        2. Change memory scope
        3. Update available tools
        4. Adjust delegation strategy
        """
        
        old_state = self.current_state
        
        # Record transition
        self.state_history.append({
            "from": old_state,
            "to": new_state,
            "reason": reason,
            "timestamp": datetime.now()
        })
        
        # Get state config
        state_config = LunaContextStateMachine.states[new_state]
        
        # 1. Swap LoRA
        await self.lora_manager.load_adapter(
            state_config["lora"]
        )
        
        # 2. Update memory scope
        self.memory_matrix.set_scope(
            state_config["memory_scope"]
        )
        
        # 3. Update available tools
        self.available_tools = state_config["tools"]
        
        # 4. Update delegation strategy
        self.delegation_strategy = state_config["delegation_strategy"]
        
        self.current_state = new_state
        
        # Optional: Announce transition
        if old_state != new_state:
            return f"[Switched to {new_state} mode]"

## Project Pipeline Manager

### **Context-Aware Project Tracking**

```python
class LunaProjectManager:
    """
    Manage projects with context awareness.
    Different views for different contexts.
    """
    
    def __init__(self):
        self.projects = {}
        self.context_views = {}
    
    async def get_projects(self, context: str) -> list[Project]:
        """
        Get projects relevant to current context.
        
        Professional: Work projects
        Personal: Life goals, hobbies
        Creative: Art projects, experiments
        Learning: Courses, studies
        """
        
        if context == "professional":
            return [p for p in self.projects.values() 
                    if p.category == "work" and p.status == "active"]
        
        elif context == "personal":
            return [p for p in self.projects.values()
                    if p.category in ["personal", "health", "relationships"]]
        
        elif context == "creative":
            return [p for p in self.projects.values()
                    if p.category in ["art", "writing", "music", "experiments"]]
        
        elif context == "learning":
            return [p for p in self.projects.values()
                    if p.category == "education"]
    
    async def delegate_task(self, task: Task, context: str):
        """
        Delegate task with context-appropriate strategy.
        """
        
        if context == "professional":
            # Work tasks: Execute immediately
            priority = self._calculate_work_priority(task)
            if priority > 0.7:
                await self._execute_immediately(task)
            else:
                await self._schedule_task(task)
        
        elif context == "personal":
            # Personal tasks: Consider timing and energy
            user_energy = await self._check_user_energy()
            if user_energy < 0.5 and task.effort > 0.6:
                await self._suggest_later(task, reason="You seem tired")
            else:
                await self._offer_help(task)
        
        elif context == "deep_work":
            # Deep work: Batch interruptions
            if task.urgency < 0.8:
                await self._defer_until_break(task)
            else:
                await self._interrupt_with_context(task)
```
