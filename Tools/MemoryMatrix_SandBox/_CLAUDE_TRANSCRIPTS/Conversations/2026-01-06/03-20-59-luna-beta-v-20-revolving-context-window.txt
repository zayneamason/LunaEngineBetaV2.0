Core causes:

1. **Context gaps** — Model doesn't have the information, so it confabulates something that *sounds right*
2. **Stale context** — Had the info earlier in conversation, but it's been pushed out of the window
3. **Blended context** — Multiple similar things in context, model mixes attributes between them
4. **Overconfidence in patterns** — Training makes certain phrasings feel "right" even when wrong
5. **No retrieval anchor** — Nothing in context to ground the response to, so it freestyles

**The window problem specifically:**

When context gets truncated (standard sliding window), the model loses track of what was established earlier. It doesn't *know* it forgot — it just generates as if it has complete information.

**How an LLM actually processes:**

**1. Tokenization**
Your message gets chopped into tokens (roughly word-chunks). "I'm getting coffee" → `["I", "'m", " getting", " coffee"]`

**2. Embedding**
Each token becomes a high-dimensional vector — a point in space. Similar meanings = closer points.

**3. Attention (this is the core mechanic)**
Every token looks at every other token and asks "how relevant are you to me?" This happens in parallel, across multiple "heads" looking for different patterns (syntax, semantics, reference, etc.)

There's no sequence in processing — it's all at once. The "context window" is just how many tokens can attend to each other simultaneously.

**4. Layers**
This attention + transformation happens repeatedly through layers. Early layers = surface patterns. Later layers = abstract meaning.

**5. Output**
Final layer produces probability distribution over all possible next tokens. Sample from that. Repeat.

---

- **No memory** — just what's in the window right now
- **No hierarchy** — all tokens are peers, weighted only by attention scores
- **No metadata** — I don't know *why* something is in context, just that it is
- **Position matters somewhat** — there's positional encoding, but attention can reach anywhere

**1. Message Arrives**

**2. System Prompt Assembly**
Before tokenization, the full prompt is constructed:
```

[Tool definitions]
[Conversation history (all prior turns)]
[Your new message]
```
This is one giant string.

**3. Tokenization**
That string → token IDs. "Hello world" → `[15496, 995]` (roughly). Each token is an integer mapping to the vocabulary.

**4. Embedding Lookup**
Token IDs → vectors. Each token becomes a high-dimensional point (e.g., 4096 dimensions for larger models).

**5. Positional Encoding**
Position information gets added — the model needs to know token order since attention is position-agnostic by default.

**6. Transformer Layers (the actual "thinking")**
For each layer (dozens of them):
```
Input → Layer Norm → Multi-Head Attention → Residual Add
      → Layer Norm → Feed Forward Network → Residual Add → Output
```
Attention is where tokens "look at" each other. FFN is where pattern transformation happens.

**7. Final Layer → Logits**
Output of last layer gets projected to vocabulary size. You get a score for every possible next token.

**8. Sampling**
Logits → probabilities (softmax) → sample one token based on temperature/top-p/etc.

**9. Repeat 6-8**
New token appends to sequence. Run through all layers again. Sample next token. Loop until stop condition.

**10. Detokenization**
Token IDs → text string. That's my response.
