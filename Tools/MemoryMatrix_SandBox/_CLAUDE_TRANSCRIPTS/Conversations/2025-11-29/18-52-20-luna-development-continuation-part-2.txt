## Magnetic Trigger System Design

The idea: Apps emit signals, Luna detects them, context loads automatically.

### Option A: Passive Detection (Current)
Luna analyzes message text for keywords.
- **Pro**: No app changes needed
- **Con**: Fuzzy, can misfire

### Option B: Explicit Context Header
Apps send a header/prefix that declares context.
```
[CONTEXT:post-generator:editing.editing_hook]
Help me punch up this hook
```
- **Pro**: Precise
- **Con**: Requires app integration

### Option C: Hub-Mediated Context
All AI calls go through Pipeline Hub. Hub tells Luna what app is calling.
```
POST /api/ai/complete
{
  "app": "post-generator",
  "state": "editing.editing_hook",
  "message": "Help me punch up this hook"
}
```
Hub injects Luna context before forwarding to Claude.
- **Pro**: Clean separation, apps don't know about Luna
- **Con**: Requires all apps to use hub

### Option D: WebSocket State Broadcast
Apps broadcast state changes to a shared channel. Luna listens.
```
App → Hub: { "app": "post-generator", "state": "editing.editing_hook" }
Hub → Luna: State updated
Luna: Loads appropriate logic file
```
- **Pro**: Real-time, decoupled
- **Con**: More infrastructure

**My recommendation: Option C (Hub-Mediated)** 

It's the cleanest. Pipeline Hub is already the central AI service. Make it the integration point:

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  Post Generator │────▶│  Pipeline Hub   │────▶│  Luna MCP       │
```

```bash

$ pipeline status      # Show what's running
$ pipeline down        # Stop all
```
