## Lock-In Coefficient: The Math

The Memory Matrix already tracks:

| Field | What It Measures |
|-------|------------------|
| `access_count` | How often retrieved |
| `accessed_at` | Last access time |
| `confidence` | How certain the info is |
| `edges` count | Graph connectivity |
| `created_at` | Age |

**Lock-in combines these:**

```python
def compute_lock_in(node: Node, graph: GraphIndex) -> float:
    """
    Compute memory importance score (0-1).
    Higher = more "locked in" to Luna's mind.
    """
    # 1. Access frequency (log scale, capped)
    access_score = min(math.log1p(node.access_count) / 5.0, 1.0)
    
    # 2. Recency (exponential decay)
    days_since_access = (now - node.accessed_at).days
    recency_score = math.exp(-days_since_access / 30.0)  # 30-day half-life
    
    # 3. Confidence
    confidence_score = node.confidence
    
    # 4. Graph centrality (how connected)
    edge_count = graph.degree(node.id)
    centrality_score = min(edge_count / 10.0, 1.0)  # Cap at 10 edges
    
    # 5. Weighted combination
    lock_in = (
        0.30 * access_score +
        0.25 * recency_score +
        0.20 * confidence_score +
        0.25 * centrality_score
    )
    
    return lock_in
```

def compute_loss_weights(batch: list[Node]) -> list[float]:
