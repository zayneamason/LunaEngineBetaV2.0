**Two distinct memory types:**

## Proposed Architecture: Library Sync System

### Component: LibrarySyncManager

```
Component: LibrarySyncManager
├─ Responsibility: Ingest, index, and sync external folders into Luna's searchable library
├─ Dependencies: MemoryMatrix (for storage), DocumentParser (for extraction)
├─ Interface:
│   ├─ add_source(path, name, options) → SourceID
│   ├─ remove_source(source_id)
│   ├─ sync_source(source_id) → SyncResult
│   ├─ sync_all() → List[SyncResult]
│   ├─ list_sources() → List[Source]
│   └─ get_source_status(source_id) → SourceStatus
├─ State:
│   ├─ sources.json (registered folder mappings)
│   └─ sync_state.json (last sync timestamps, file hashes)
└─ Failure modes:
    ├─ File parse failure → log + skip, don't fail whole sync
    ├─ Missing folder → mark source as "disconnected"
    └─ Conflicting chunk IDs → deterministic hash resolution
```

**On versioning integration—here's my read:**

The file watcher detects *file system* changes, but versioning gives you *content* history. Two layers:

1. **Change Detection** (watcher): "This file was modified"
2. **Version Tracking** (AI-BRARIAN): "Here's what changed, here's the diff, here's the new chunks to index"

This lets you do smart re-indexing:
- File touched but content identical? Skip.
- Small edit in one section? Re-chunk only that section, preserve chunk IDs for unchanged parts.
- Major rewrite? Full re-index, mark old chunks as superseded (not deleted—Memory Matrix philosophy).

**The versioning tie-in also enables:**
- "What did the Mars packing list say *before* they updated it?"
- Rollback if a sync corrupts something
- Audit trail for Luna: "I learned this from v2 of that doc, but v3 changed it"

```python
class AIBrarian:
    """
    The conductor. Coordinates all subsystems.
    """
    
    def __init__(self, data_dir: str):
        # Wire everything together
        self.sources = SourceManager(data_dir)
        self.versions = VersionTracker(data_dir)
        self.router = DocumentRouter()
        self.chunker = SemanticChunker(llm_provider=HybridProvider())
        self.index = LibraryIndex(memory_matrix)  # TBD
    
    def add_source(self, path: str, name: str) -> Source:
        """Register a folder for syncing."""
        return self.sources.register(path, name)
    
    def sync_source(self, source_id: str) -> SyncResult:
        """
        THE MAIN WORKFLOW:
        1. Get all files from source path
        2. For each file:
           a. Check if changed (version tracker)
           b. If changed: parse → chunk → index
           c. Record new version
        3. Handle deletions
        4. Return summary
        """
        source = self.sources.get(source_id)
        
        for file_path in self._get_files(source.path):
            # Read file
            content = file_path.read_bytes()
            new_hash = self.versions.compute_hash(content)
            
            # Skip if unchanged
            if not self.versions.has_changed(source_id, str(file_path), new_hash):
                continue
            
            # Parse
            result = self.router.parse(file_path)
            if not result.success:
                continue
            
            # Chunk
            chunks = self.chunker.chunk(result.document, source_id, new_hash)
            
            # Index into Memory Matrix
            self.index.index_chunks(chunks.chunks)
            
            # Record version
            self.versions.record_version(source_id, str(file_path), content)
        
        return SyncResult(...)
    
    def search(self, query: str) -> List[SearchResult]:
        """Unified search across library + session memory."""
        return self.index.search(query)
```
