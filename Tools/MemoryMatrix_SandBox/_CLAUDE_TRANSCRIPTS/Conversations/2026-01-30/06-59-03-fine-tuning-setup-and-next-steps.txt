[ASSISTANT] 2026-01-30T07:44:03.568281Z
 Yeah, they can. MLX and most inference frameworks don't care where the model lives — they just need a path.

**The tradeoff is speed:**

| Storage | Sequential Read | Random Read | Model Load Time |
|---------|-----------------|-------------|-----------------|
| Internal SSD (NVMe) | ~3-7 GB/s | ~500K+ IOPS | Fast (~2-5s for 3B) |
| External SSD (USB-C/TB3) | ~1-2 GB/s | ~50-100K IOPS | Slower (~5-15s) |
| External HDD (spinning) | ~100-150 MB/s | ~100-200 IOPS | Painful (~30-60s+) |

**What matters:**
- **Model loading** = sequential read (one-time cost per session)
- **Inference** = model is in RAM after load (storage speed irrelevant)

So if your external is an SSD over Thunderbolt/USB-C, the only penalty is a longer cold start. Once loaded, inference runs at full speed from unified memory.

---

**To use models from external:**

```python
# Point mlx-lm at external path
from mlx_lm import load

model, tokenizer = load("/Volumes/YourDrive/models/Qwen2.5-7B-Instruct")
```

Or symlink into the HuggingFace cache:
```bash
ln -s /Volumes/YourDrive/models/Qwen--Qwen2.5-7B-Instruct \
      ~/.cache/huggingface/hub/models--Qwen--Qwen2.5-7B-Instruct
```
