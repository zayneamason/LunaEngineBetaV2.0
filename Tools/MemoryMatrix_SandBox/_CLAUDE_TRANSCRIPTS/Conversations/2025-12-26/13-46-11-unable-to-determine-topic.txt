4. **Audio playback**: Variable

Stream the response as it generates:
- User finishes speaking

## Industry Standard Approaches

### 1. **Two-Model Architecture** (Most Common)
**What everyone does:**
- **Fast model** for real-time conversation (Haiku, GPT-3.5-Turbo, or specialized voice models)
- **Smart model** runs in background, updating context

**Examples:**
- **ChatGPT Voice**: Uses Whisper (STT) → GPT-4o-mini for speed → streams response
- **Google Assistant/Alexa**: Lightweight on-device model → cloud fallback
- **Anthropic's own demos**: Likely Haiku for voice, Opus for complex queries

```
User speaks
    ↓
STT (300ms)
    ↓
Haiku responds (2-3s) ← User hears this
    ↓
Background: Sonnet enriches memory/context for next turn
```

### 2. **Aggressive Context Pruning for Voice**
Voice-specific context is **drastically smaller** than desktop:

**Desktop Chat**: "Give me everything Luna knows"
- 2000-3000+ tokens
- Rich personality, full history
- 15-20+ second thinking time is acceptable

**Voice Mode**: "Give me just enough to sound like Luna"
- 300-800 tokens MAX
- Core personality traits only
- Recent conversation only (last 3-5 turns)
- No deep memory retrieval unless explicitly asked

**Real example** (ChatGPT Voice):
```
System: You are ChatGPT, a helpful assistant. [~50 tokens]
Recent context: [Last 3 turns, ~200 tokens]
Total: ~250 tokens
Response time: 1-2 seconds
```

### 3. **Streaming is Non-Negotiable**
**Everyone streams for voice:**
- First token in 1-2 seconds
- TTS starts immediately on first sentence
- User hears response while model is still thinking

**Your current setup**: Waiting for full response before TTS
**Industry standard**: Stream → TTS each sentence as it arrives

### 4. **Specialized Voice Models**
Many companies use **purpose-built voice models**:
- OpenAI: GPT-4o (optimized for voice)
- Google: Gemini Live uses voice-optimized variant
- Anthropic: Likely has internal voice-tuned models

These are:
- Faster (optimized for low-latency)
- Smaller context windows
- Trained specifically for conversational flow

### 5. **Hybrid Intelligence**
**Pattern**: Fast response now, smart enrichment later

```
Turn 1: Haiku responds in 2s with basic Luna personality
        ↓ (background)
        Sonnet analyzes conversation, updates personality state

Turn 2: Haiku responds in 2s, now with enriched context from Turn 1
        ↓ (background)  
        Sonnet continues learning

Turn N: Haiku is responding with increasingly rich context,
```
