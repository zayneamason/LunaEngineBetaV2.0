# Demo: Kinoni Eclissi Database

This demo shows Luna's knowledge architecture in action -- not retrieval, but connected knowledge. The difference between a search engine and a memory.

## What You're Looking At

A seeded Eclissi database for the Nakaseke Restoration Collective in central Uganda. 13 entities, 9 relationships, 11 knowledge nodes across five domains: elder wisdom, governance, land stewardship, youth programs, and funding history.

This is what six months of community use looks like as structured data.

## Key Demo Moments

### 1. The Funding Gap

Ask: **"What did we learn from the funding gap?"**

Watch how Luna connects: the 2023 funding crisis (GreenFutures grant ended, Resilience Fund delayed three months) to Jjajja Nakato's recollection of a 1990s drought where the community pooled grain reserves. The community contribution model they activated wasn't invented -- it was remembered. Luna surfaces the precedent.

### 2. The Hillside Erosion

Ask: **"How do we resolve the hillside dispute?"**

Luna connects: Elder Musoke's Three Hearings governance protocol (anger, truth, wisdom) to the unauthorized tree clearing on the northern hillside. Then links to Nakato's planting knowledge and the youth scout monitoring program. Multiple elders' knowledge converging on one problem through the graph.

### 3. A Person's Full Contribution

Ask: **"What has Tendo contributed?"**

Luna builds a contribution graph: the youth transition protocol Tendo helped design, digital skills training, governance documentation, wetland monitoring setup. A complete picture of one person's impact across multiple domains -- the kind of institutional memory that usually walks out the door when someone leaves.

### 4. Governance in Practice

Ask: **"What governance protocols do we have?"**

Luna returns: the Three Hearings principle, the NGO consent protocol (no external data collection without council permission), the youth transition protocol, and the water access resolution. Each one linked to the real event that created it. Governance as living practice, not policy documents.

### 5. Connected Knowledge

Ask: **"Who should I talk to about land stewardship?"**

Luna doesn't just return a name. It shows you Elder Kabuye (land and ecology), links to the wetland restoration results (40% papyrus recovery, three bird species returned), connects to Ssemanda's field crew, and surfaces Nakato's planting season teachings. A web of expertise, not a directory.

## How to Run

The database loads from seed data in this folder. The LUNA-Teaser-1.mp4 video provides additional visual context.

For a live demo with Luna's full conversational engine, run:

```
./scripts/launch_luna.sh
```

Then open the Luna sidebar in the Master Index Sheet, or connect directly to the local API.

## The Point

The power isn't in any single answer. It's in the connections between answers. A search engine finds documents. Luna finds relationships between knowledge -- across people, across time, across domains.

That's what sovereign memory looks like.
