Scout & Overdrive

Blockage Detection · Adaptive Retrieval · Cooldown Cycle
The Flow
1
DIRECTOR GENERATES DRAFT
Normal path. Director assembles context, generates response via local or cloud inference. This is the current behavior — fast, personality-rich, but sometimes empty-handed.
2
SCOUT INSPECTS DRAFT
Before the response is delivered, the Scout agent reads the draft. Pattern-matches against known blockage signatures. Evaluates depth of supporting evidence. Decides: pass through or trigger Overdrive.
3
OVERDRIVE ACTIVATES
Scout determines blockage depth → selects Overdrive tier → adjusts retrieval weights: token budget, source count, timeout duration, search chain breadth. Re-runs retrieval with expanded parameters. Director re-generates with enriched context. Cooldown timer starts.
4
DELIVER RESPONSE
Either the original draft (if Scout passed it) or the re-generated response (if Overdrive fired). Debug metadata includes: route decision, blockage type detected, Overdrive tier used, cooldown state.
What the Scout Detects
SURRENDER
"i don't have any memories"
"i don't know about"
"not in my memory"
"tell me more"
"can you fill me in"
Luna admits she has no information. High confidence blockage — retrieval returned empty or was never triggered.
SHALLOW RECALL
"details are a bit fuzzy"
"i don't have much more"
"beyond that, i'm afraid"
"that's about all i can say"
"from what i can recall" + short response
Luna found something but not enough. Retrieval hit but token budget was too tight or search was too narrow.
DEFLECTION
"what can you tell me about"
"i'd love to hear your thoughts"
"can you share more"
[asking user to teach her about her own project]
Luna reverses the question — asks the user to provide context that exists in her own memory. Variant of surrender.
HEDGING
"it seems like" + no specifics
"if i recall correctly" + vague claim
"i believe" + unanchored statement
response < 100 chars on a knowledge query
Luna is guessing. Low confidence language without supporting evidence from retrieved context.

Cooldown Cycle
Overdrive Energy — 60s Default Cooldown
COOLING (0-20s)
Overdrive just fired. No re-triggers. Normal retrieval only. Scout still monitors but can't activate.
WARMING (20-50s)
Partial recovery. Scout can trigger Tier 1 only. Higher tiers still locked.
READY (50s+)
Full charge. All tiers available. Scout has full authority to trigger any depth of Overdrive.

Uses What Already Exists
DETECTION
Pattern matching on draft response. Same approach as existing surrender_intercept in persona_adapter.py. Extended with shallow recall + deflection + hedging patterns.
RETRIEVAL
SearchChainConfig already supports variable token budgets, source lists, and limits. Overdrive just passes different config parameters to run_search_chain().
AGENTIC
AgentLoop exists for Tier 3. ToolRegistry has memory_tools, dataroom_tools, file_tools registered. No new tools needed.
ACTOR FRAMEWORK
Scout extends BaseActor (src/luna/actors/base.py). Same pattern as Director, Matrix, Scribe, Librarian. Registered via engine.register_actor().
